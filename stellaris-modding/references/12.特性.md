# 特性（Traits）のMod作成ガイド

このドキュメントでは、Stellaris における種族特性とリーダー特性のMod作成方法について解説します。

---

## ファイル構造

| 種類 | パス |
|------|------|
| 特性 | `common/traits/*.txt` |

---

## 種族特性とリーダー特性の比較

| プロパティ | 種族特性 | リーダー特性 |
|-----------|---------|-------------|
| `cost` | 帝国作成時の特性ポイントコスト | リーダーの特性スロットコスト |
| `initial` | 帝国作成時に選択可能か | リーダー生成時に付与可能か |
| `randomized` | ランダム種族生成で選択されるか | 新規リーダーにランダム付与されるか |
| `modification` | 遺伝子改変で追加/削除可能か | 通常は `no` |
| `modifier` | POP・軍隊・リーダーに適用 | リーダーの配属先スコープに適用 |

---

## 種族特性

### データ構造

```pdx
<trait_key> = {
    cost = <int>                              # 特性ポイントコスト
    category = <normal/origin/special>        # カテゴリ
    
    # 選択可否
    initial = <yes/no>                        # 帝国作成時に選択可能（デフォルト: yes）
    randomized = <yes/no>                     # ランダム種族生成で選択（デフォルト: yes）
    
    # 遺伝子改変
    modification = <yes/no>                   # 改変で追加/削除可能（デフォルト: yes）
    advanced_trait = <yes/no>                 # 上級特性技術が必要
    
    # アイコン
    icon = "<gfx_path>"                       # デフォルト: gfx/interface/icons/traits/<trait_key>.dds
    
    # 相互排他
    opposites = { trait_xx trait_yy }
    
    # アーキタイプ・種族クラス制限
    allowed_archetypes = { BIOLOGICAL LITHOID MACHINE ROBOT }
    species_class = { MAM REP }               # 種族クラス制限（オプション）
    allowed_planet_classes = { pc_desert }    # 母星タイプ制限（オプション）
    
    # 前提技術
    prerequisites = { tech_xxx }
    
    # 追加/削除条件（種族スコープ、FROM = 帝国）
    species_potential_add = { <conditions> }
    species_possible_remove = { <conditions> }
    species_possible_merge_add = { <conditions> }
    species_possible_merge_remove = { <conditions> }
    
    # 修飾子
    modifier = {
        <modifiers>
    }
    
    # 条件付きPOP修飾子（POPスコープ）
    triggered_pop_modifier = {
        potential = { <conditions> }
        <modifiers>
    }
    
    # 成長・組立・減少時の惑星修飾子
    growing_modifier = { <modifiers> }
    assembling_modifier = { <modifiers> }
    declining_modifier = { <modifiers> }
    
    # 特殊フラグ
    sapient = <yes/no>                        # 準知性体特性の場合 no
    improves_leaders = <yes/no>               # リーダー強化ツールチップ表示
    immortal_leaders = <yes/no>               # 不死リーダー
    forced_happiness = <yes/no>               # 幸福度の影響を無視
    infertile = <yes/no>                      # 不妊（成長不可）
    
    # リーダー年齢範囲
    leader_age_min = <int>
    leader_age_max = <int>
    
    # その他
    sorting_priority = <int>                  # 表示ソート順
    potential_crossbreeding_chance = <float>  # 異種交配の継承確率
    slave_cost = { <resource> = <int> }       # 奴隷市場での価格加算
    
    # AIウェイト
    ai_weight = {
        weight = <int>
        modifier = { ... }
    }
    
    # タグ（内部分類用）
    tags = { organic positive pop_output food }
    
    # カスタムツールチップ
    custom_tooltip = <loc_key>
    custom_tooltip_with_modifiers = <loc_key>
}
```

### 重要プロパティ

#### allowed_archetypes

種族アーキタイプによる制限：

| アーキタイプ | 説明 |
|-------------|------|
| `BIOLOGICAL` | 有機生命体 |
| `LITHOID` | 岩石種族 |
| `MACHINE` | 機械知性 |
| `ROBOT` | ロボット |
| `PRESAPIENT` | 準知性体 |

#### category

| カテゴリ | 説明 |
|---------|------|
| `normal` | 通常の特性（ピック可能） |
| `origin` | オリジン由来の特性 |
| `special` | 特殊特性（イベント等で付与） |

#### triggered_pop_modifier

条件付きで適用される修飾子。ツールチップに表示されないため、`custom_tooltip` と併用：

```pdx
triggered_pop_modifier = {
    potential = {
        has_job = miner
    }
    planet_jobs_minerals_produces_mult = 0.10
}
custom_tooltip = TRAIT_MINING_BONUS_TOOLTIP
```

### バニラ例: 独創的

```pdx
trait_ingenious = {
    cost = 2
    category = normal
    species_possible_add = {
        can_add_genetic_traits = yes
    }
    species_possible_remove = {
        OR = {
            can_remove_beneficial_genetic_traits = yes
            can_remove_vocational_genetic_traits = yes
        }
    }
    allowed_archetypes = { BIOLOGICAL LITHOID }
    tags = { organic positive pop_output energy }
    modifier = {
        technician_jobs_bonus_workforce_mult = 0.15
    }
    slave_cost = {
        energy = 500
    }
}
```

### バニラ例: 準知性体特性

```pdx
trait_presapient_proles = {
    cost = 0
    opposites = { "trait_presapient_natural_intellectuals" }
    initial = no
    sapient = no
    species_possible_remove = {
        can_remove_presapient_genetic_traits = yes
    }
    allowed_archetypes = { BIOLOGICAL PRESAPIENT LITHOID }
    tags = { organic positive pop_output mineral food research presapient }
    modifier = {
        army_damage_mult = 0.10
        worker_and_simple_drone_cat_bonus_workforce_mult = 0.10
        researcher_jobs_bonus_workforce_mult = -0.15
    }
}
```

## リーダー特性

### データ構造

```pdx
<trait_key> = {
    # 特性タイプ
    leader_trait_type = <veteran/destiny/negative>  # オプション
    
    # アイコン（inline_script使用が一般的）
    inline_script = {
        script = trait/icon
        CLASS = commander
        ICON = GFX_leader_trait_xxx
        RARITY = common/veteran/paragon
        COUNCIL = yes/no
        TIER = 1/2/3/none
    }
    
    # 対象リーダークラス
    leader_class = { commander scientist official }
    leader_trait = all                        # 古い形式
    
    # 生成条件（リーダースコープ、FROM = 帝国）
    leader_potential_add = { <conditions> }
    
    # 前提技術
    prerequisites = { tech_xxx }
    
    # 相互排他
    opposites = { trait_xx trait_yy }
    
    # 特性の置き換え（ティア昇格）
    replace_traits = { leader_trait_xxx }
    
    # 修飾子タイプ別
    modifier = { <modifiers> }                # 汎用（リーダーの配属先に適用）
    fleet_modifier = { <modifiers> }          # 艦隊指揮時
    army_modifier = { <modifiers> }           # 軍隊指揮時
    planet_modifier = { <modifiers> }         # 総督の惑星
    sector_modifier = { <modifiers> }         # 総督のセクター
    councilor_modifier = { <modifiers> }      # 評議員時
    self_modifier = { <modifiers> }           # リーダー自身のみ
    
    # 条件付き修飾子
    triggered_modifier = {
        potential = { <conditions> }
        <modifier_type> = { <modifiers> }
    }
    triggered_self_modifier = {
        potential = { <conditions> }
        <modifiers>
    }
    
    # 取得時エフェクト
    on_gained_effect = { <effects> }
    
    # AI設定
    ai_categories = { engineering physics society survey }
    ai_weight = { weight = <int> }
    
    # 選択ウェイト
    selectable_weight = {
        weight = <int>
        modifier = { ... }
    }
    
    # 特殊フラグ
    initial = <yes/no>
    randomized = <yes/no>
    immortal_leaders = <yes/no>
    hide_age = <yes/no>                       # 年齢非表示（支配者のみ）
    force_councilor_trait = <yes/no>          # 評議員専用
    ethic_destiny_trait = <yes/no>            # 倫理に基づくデスティニー特性
    
    # カスタムツールチップ
    custom_tooltip = <loc_key>
    custom_tooltip_with_modifiers = <loc_key>
    
    # 背景アイコン
    background_icon = GFX_leader_background_destiny_1
}
```

### リーダークラス

| クラス | 説明 |
|--------|------|
| `commander` | 司令官（提督/将軍/総督） |
| `scientist` | 科学者 |
| `official` | 官僚 |

### 特性タイプ（leader_trait_type）

| タイプ | 説明 |
|--------|------|
| `veteran` | ベテラン特性（経験で獲得） |
| `destiny` | デスティニー特性（Paragon DLC） |
| `negative` | ネガティブ特性 |

### 修飾子の適用範囲

| 修飾子タイプ | 適用範囲 |
|-------------|---------|
| `modifier` | 艦隊/軍隊/惑星など配属先 |
| `fleet_modifier` | 艦隊指揮時のみ |
| `army_modifier` | 軍隊指揮時のみ |
| `planet_modifier` | 総督の直轄惑星 |
| `sector_modifier` | 総督のセクター全体 |
| `councilor_modifier` | 評議員として任命時 |
| `self_modifier` | リーダー自身（経験値・寿命等） |

### バニラ例: 攻撃的

```pdx
leader_trait_aggressive = {
    inline_script = {
        script = trait/icon
        CLASS = commander
        ICON = GFX_leader_trait_aggressive
        RARITY = common
        COUNCIL = no
        TIER = 1
    }
    fleet_modifier = {
        ship_fire_rate_mult = 0.05
        ship_weapon_damage = 0.05
    }
    leader_class = { commander }
    opposites = {
        leader_trait_cautious
    }
    selectable_weight = {
        weight = @class_trait_weight
        inline_script = paragon/pilot_weight_mult
    }
}
```

### バニラ例: デスティニー特性

```pdx
leader_trait_master_gunner = {
    leader_trait_type = destiny
    inline_script = {
        script = trait/icon
        CLASS = commander
        ICON = GFX_leader_trait_master_gunner
        RARITY = paragon
        COUNCIL = no
        TIER = none
    }
    on_gained_effect = {
        if = {
            limit = { exists = owner }
            owner = {
                country_event = { id = timeline.50 }
            }
        }
    }
    fleet_modifier = {
        ship_weapon_range_mult = 0.3
        ship_tracking_mult = 0.3
        ship_accuracy_mult = 0.3
    }
    leader_potential_add = {
        has_paragon_dlc = yes
        has_trait = subclass_commander_admiral
    }
    leader_class = { commander }
    background_icon = GFX_leader_background_destiny_1
}
```

### バニラ例: ネガティブ特性

```pdx
leader_trait_armchair_commander = {
    leader_trait_type = negative
    inline_script = {
        script = trait/icon_negative
        ICON = "GFX_leader_trait_armchair_commander"
        COUNCIL = no
        TIER = 1
    }
    army_modifier = {
        army_morale = -0.05
        army_damage_mult = -0.05
    }
    leader_potential_add = {
        trait_is_crisis_faction_check = no
        is_gestalt = no
    }
    leader_class = { commander }
    opposites = {
        leader_trait_butcher
    }
}
```

## 特性のティア昇格

`replace_traits` を使用して、特性を上位ティアに昇格させます：

```pdx
leader_trait_aggressive_2 = {
    replace_traits = { leader_trait_aggressive }
    leader_trait_type = veteran
    inline_script = {
        script = trait/icon
        CLASS = commander
        ICON = GFX_leader_trait_aggressive
        RARITY = veteran
        COUNCIL = no
        TIER = 2
    }
    fleet_modifier = {
        ship_fire_rate_mult = 0.10
        ship_weapon_damage = 0.10
    }
    leader_class = { commander }
}
```

## inline_script の使用

現代のバニラ特性は `inline_script` を使用してアイコン定義を簡略化：

```pdx
inline_script = {
    script = trait/icon
    CLASS = commander        # commander/scientist/official/leader
    ICON = GFX_leader_trait_xxx
    RARITY = common          # common/veteran/paragon/free_or_veteran
    COUNCIL = yes            # 評議員特性か
    TIER = 1                 # 1/2/3/none
}
```

## 実装上の注意

### 種族特性

- `cost = { base = ... modifier = { ... } }` を使う場合、帝国作成画面やランダム生成では country scope が無いので `base` を必ず妥当な値にする。
- 帝国作成時の選択制限は `allowed_origins` / `forbidden_origins` / `allowed_civics` / `forbidden_civics` / `allowed_ethics` / `forbidden_ethics` で制御できる。
- `allowed_planet_classes` と `species_class` にはそれぞれ `*_override` があり、通常制限を例外的に無視させられる。
- `portrait_override` はアーキタイプや種族クラスを跨いだ例外許可に使える。
- `triggered_planet_growth_habitability_modifier`、`triggered_pop_group_modifier`、`triggered_planet_pop_group_modifier_for_*` は自動ツールチップに出ないため、`custom_tooltip` / `custom_tooltip_with_modifiers` を併用する。

### リーダー特性

- イベント配布や固定配布専用の特性は `initial = no` と `randomized = no` を明示して、通常生成テーブルへ混ぜない。
- 支配者作成時に選ばせたい特性は `starting_ruler_trait = yes` を使う。
- 評議会在席時の国家補正を条件付きで切り替えたい場合は `triggered_councilor_modifier` を使う。`force_councilor_trait = yes` は councilor_modifier を持たない特性も評議員特性として扱わせるための補助。
- リーダー特性の `triggered_modifier` 系は `THIS = leader`、`FROM = 実際の適用先scope` で評価される。艦隊・惑星・評議会で同じ条件式を流用するときは `from` の中身が変わる前提で書く。
