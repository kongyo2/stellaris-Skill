# アノマリー (Anomalies)

## 概要

アノマリーは、科学船が天体を調査した際に確率で発見される特別なイベントです。ゲームに物語的な深みを与え、探索の面白みを増す重要な要素です。

アノマリーのModdingは主に2つの要素から構成されます：

| 要素 | 説明 | 配置場所 |
|:--|:--|:--|
| **アノマリーカテゴリ** | 発見されるアノマリーの定義（難易度、説明文、出現条件） | `/common/anomalies/*.txt` |
| **アノマリーイベント** | 調査完了後に発生するイベント | `/events/*.txt` |

---

## アノマリーの研究

アノマリーにはレベル（難易度）が1から10まで設定されており、科学者のスキルレベルに応じて研究時間が変化します。

### 研究時間と難易度の目安

| スキル差 | 研究時間(日) | 相対難易度 |
|:--:|:--:|:--|
| +9 | 20 | 朝飯前 |
| +7 | 30 | 退屈 |
| +5 | 45 | 極めて簡単 |
| +3 | 65 | 簡単 |
| +1 | 100 | 日常的 |
| 0 | 120 | 日常的 |
| -2 | 300 | 挑戦的 |
| -4 | 720 | 困難 |
| -6 | 1440 | 極めて困難 |
| -8 | 2880 | 地獄 |

> [!TIP]
> Mod制作時は、アノマリーの重要度や報酬に応じて適切なレベルを設定しましょう。レベル1-3は序盤向け、レベル6以上は重要なイベント向けです。

## アノマリーの発生メカニズム

天体が初めて調査されるたびに、ゲームはアノマリー発生の判定を行います。

### 発生確率

| 定数名 | デフォルト値 | 説明 |
|:--|:--:|:--|
| `ANOMALY_SPAWN_CHANCE` | 0.05 (5%) | 基本発生確率 |
| `ANOMALY_SPAWN_CHANCE_INCREMENT` | 0.005 (0.5%) | 失敗時の増加確率 |

これらは `common/defines/00_defines.txt` で定義されています。

### 発生条件

カテゴリが選択されるには以下の条件を満たす必要があります：

1. そのアノマリーがまだ調査されずにマップ上に存在していない
2. `should_ai_use` / `should_ai_and_humans_use` の設定に基づき発見可能
3. `max_once` / `max_once_global` の制約を満たしている
4. `spawn_chance` ブロック内のトリガー条件を満たしている
5. `null_spawn_chance` のチェックを通過する

## アノマリーカテゴリの定義

ファイルは `/common/anomalies/` に `.txt` ファイルとして作成します。

### 基本構造

```pdx
# /common/anomalies/my_anomaly_categories.txt

my_anomaly_category_id = {
    # === 基本設定 ===
    desc = "my_anomaly_category_id_desc"  # 状況ログの説明文
    picture = GFX_evt_asteroid_field       # 表示画像
    level = 3                              # 難易度レベル (1-10)

    # === 出現制御 ===
    should_ai_use = no                     # AIが発見可能か (デフォルト: no)
    should_ai_and_humans_use = no          # 両方が発見可能 (should_ai_useを上書き)
    max_once = yes                         # 帝国ごとに一度のみ (デフォルト: no)
    max_once_global = no                   # ゲーム全体で一度のみ (デフォルト: no)
    null_spawn_chance = 0.0                # 抽選後に発生しない確率 (0.0-1.0)

    # === 出現確率 ===
    spawn_chance = {
        base = 100                         # 基本の重み
        modifier = {
            factor = 0                     # 0で出現不可
            this = { is_asteroid = no }    # 小惑星でなければ出現しない
        }
        modifier = {
            add = 10                       # 重みを加算
            leader = { has_skill_level >= 5 }
        }
    }

    # === 発見時エフェクト (任意) ===
    # スコープ: this/root = 天体, from = 科学船
    on_spawn = {
        root = { set_planet_flag = my_anomaly_spawned }
    }

    # === 調査成功時のイベント ===
    on_success = {
        75 = {
            max_once = yes                 # この結果は帝国ごとに一度
            modifier = {
                factor = 1.5
                leader = { has_trait = leader_trait_meticulous }
            }
            anomaly_event = my_event.1
        }
        25 = anomaly_event = my_event.2    # ショートハンド
        1 = my_event.3                     # さらに短いショートハンド
    }
}
```

### 動的な説明文

`desc` はトリガーを使って動的に変更できます：

```pdx
my_anomaly_category = {
    desc = {
        trigger = { leader = { has_trait = leader_trait_carefree } }
        text = "my_anomaly_desc_carefree"  # 「大雑把」特性持ちの場合
    }
    desc = {
        trigger = { leader = { has_skill_level >= 5 } }
        text = "my_anomaly_desc_skilled"   # スキル5以上の場合
    }
    desc = "my_anomaly_desc"               # デフォルト
    # ...
}
```

## 主要なキーの詳細

### 基本設定

| キー | 型 | 説明 |
|:--|:--|:--|
| `desc` | string/block | 状況ログの説明文キー。トリガーで動的変更可能 |
| `picture` | string | 状況ログに表示される画像 (`GFX_` プレフィックス) |
| `level` | int (1-10) | 難易度。高いほど調査時間が長い |

### 出現制御

| キー | 型 | デフォルト | 説明 |
|:--|:--|:--:|:--|
| `should_ai_use` | bool | no | AIがこのアノマリーを発見できるか |
| `should_ai_and_humans_use` | bool | no | AI/プレイヤー両方が発見可能（上書き） |
| `max_once` | bool | no | 帝国ごとに一度のみ出現 |
| `max_once_global` | bool | no | ゲーム全体で一度のみ出現 |
| `null_spawn_chance` | float | 0.0 | 抽選後に「何も発見しなかった」確率 |

### spawn_chance ブロック

| キー | 説明 |
|:--|:--|
| `base` | 基本となる重み |
| `modifier` | 条件に応じて `add`(加算) または `factor`(乗算) で確率を変動 |

**スコープ:**
- `this` / `root` = 調査対象の天体
- `from` = 科学船
- `leader` = 科学船の科学者

### on_success ブロック

重み付けされたランダムリストから一つが選ばれます：

```pdx
on_success = {
    # 重み75で my_event.1 を発生
    75 = {
        max_once = yes                # この結果は帝国ごとに一度 (デフォルトyes)
        max_once_global = no          # ゲーム全体制限 (デフォルトno)
        modifier = { ... }            # 発生確率の変動
        anomaly_event = my_event.1
    }
    # 重み25で my_event.2 を発生
    25 = anomaly_event = my_event.2
    # ショートハンド形式
    1 = my_event.3
}
```

> [!IMPORTANT]
> `on_success` 内のブロックでは `max_once` のデフォルトが `yes` です。同じイベントを複数回発生させたい場合は明示的に `max_once = no` を設定してください。

## アノマリーイベントの定義

ファイルは `/events/` に `.txt` ファイルとして作成します。実質的には `ship_event` と同じです。

### 基本構造

```pdx
# /events/my_anomaly_events.txt

namespace = my_event

ship_event = {
    id = my_event.1
    title = "my_event.1.title"
    desc = "my_event.1.desc"
    picture = GFX_evt_mining_station
    show_sound = event_ship_bridge
    location = from                    # カメラを天体にズーム

    is_triggered_only = yes            # アノマリーイベントでは必須

    # スコープ: root = 科学船, from = 天体

    immediate = {
        from = {                       # 天体へスコープ変更
            clear_deposits = yes       # 既存の鉱床をクリア
        }
    }

    option = {
        name = "my_event.1.option.a"
        from = {
            set_deposit = d_engineering_2
        }
    }

    option = {
        name = "my_event.1.option.b"
        # 何もしない選択肢
    }
}
```

### スコープの違い

| 呼び出し方法 | root/this | from | fromfrom |
|:--|:--|:--|:--|
| `anomaly_event` | 科学船 | 天体 | - |
| `ship_event` (直接) | 科学船 | 科学船 | 天体 |

> [!CAUTION]
> 混乱を避けるため、特別な理由がない限りは `anomaly_event` を使用してください。

### 重要なポイント

1. **`is_triggered_only = yes`**: アノマリーイベントでは必須
2. **`location = from`**: カメラがアノマリーの天体にズームして没入感向上
3. **天体操作**: `from = { ... }` でスコープを切り替えて操作

## 実践的なサンプル

### 小惑星専用アノマリー

```pdx
# /common/anomalies/my_asteroid_anomaly.txt

my_crashed_ship_category = {
    desc = "my_crashed_ship_desc"
    picture = GFX_evt_crashed_station
    level = 3

    spawn_chance = {
        modifier = {
            add = 3
            is_asteroid = yes
        }
        modifier = {
            factor = 0
            from = { has_any_strategic_resource = yes }  # 戦略資源ありなら除外
        }
    }

    on_success = {
        1 = {
            modifier = {
                factor = 0
                from = { has_any_strategic_resource = yes }
            }
            anomaly_event = my_anomaly.1
        }
        1 = my_anomaly.2
    }
}
```

### ガス巨星専用アノマリー

```pdx
life_gasgiant_category = {
    desc = "life_gasgiant_category_desc"
    picture = GFX_evt_atmospheric_flight
    level = 2

    spawn_chance = {
        modifier = {
            add = 3
            is_planet_class = pc_gas_giant
            has_any_strategic_resource = no
        }
    }

    max_once = yes

    on_success = {
        1 = anomaly.300
        1 = anomaly.6845
    }
}
```

### 居住可能惑星アノマリー

```pdx
HBL_LUSH_PLANET_1_CAT = {
    desc = "HBL_LUSH_PLANET_1_DESC"
    picture = "GFX_evt_medieval_alien_civilization"
    level = 4

    spawn_chance = {
        modifier = {
            add = 3
            OR = {
                is_planet_class = pc_continental
                is_planet_class = pc_tropical
            }
        }
    }

    on_success = {
        1 = anomaly.85
        1 = anomaly.90
        0 = {
            modifier = {
                add = 1
                owner = {
                    OR = {
                        has_country_flag = human_1
                        has_country_flag = human_2
                        is_human_species = yes
                    }
                    NOT = { has_country_flag = found_voyager }
                }
            }
            anomaly_event = anomaly.110
        }
    }
}
```

### ゲシュタルト帝国除外

```pdx
on_crashed_ship_cat = {
    desc = "on_crashed_ship_desc"
    picture = GFX_evt_physics_cache
    level = 2

    spawn_chance = {
        modifier = {
            add = 3
            is_asteroid = yes
        }
        modifier = {
            factor = 0
            from.owner = { is_gestalt = yes }  # ゲシュタルトは除外
        }
    }

    on_success = anomaly.7030
}
```

### グローバル一回限りのレアアノマリー

```pdx
signal_gasgiant_category = {
    desc = "signal_gasgiant_category_desc"
    picture = GFX_evt_atmospheric_flight
    level = 5

    spawn_chance = {
        modifier = {
            add = 1                     # 低い重み = レア
            is_planet_class = pc_gas_giant
        }
    }

    max_once_global = yes               # ゲーム全体で一度のみ

    on_success = anomaly.5000
}
```

## よく使うトリガー条件

### 天体タイプ

```pdx
is_asteroid = yes/no
is_planet_class = pc_gas_giant
is_planet_class = pc_toxic
is_planet_class = pc_frozen
is_planet_class = pc_barren
is_planet_class = pc_barren_cold
is_planet_class = pc_molten
habitable_planet = yes
colonizable_planet = yes
uninhabitable_regular_planet = yes
is_star = no
is_moon = yes/no
is_ringworld = no
```

### 資源チェック

```pdx
has_any_strategic_resource = yes/no
has_deposit_for = shipclass_mining_station
has_deposit_for = shipclass_research_station
num_minerals > 0
```

### 所有者チェック

```pdx
from.owner = { is_gestalt = yes }
from.owner = { is_machine_empire = yes }
from.owner = { has_ethic = ethic_gestalt_consciousness }
from.owner = { is_ai = no }
from.owner = { has_country_flag = some_flag }
owner = { has_technology = tech_mine_zro }
```

### 科学者チェック

```pdx
leader = { has_skill_level >= 5 }
leader = { has_trait = leader_trait_meticulous }
from.leader = { species = { has_trait = trait_aquatic } }
```

### その他

```pdx
years_passed > 10
has_global_flag = some_global_flag
has_star_flag = some_star_flag
```

## ファイル命名規則

バニラゲームでは以下の命名パターンが使われています：

| ファイル名 | 内容 |
|:--|:--|
| `00_anomaly_categories.txt` | 基本アノマリー |
| `01_anomaly_categories.txt` | 追加アノマリー |
| `9X_anomaly_categories_*.txt` | DLC固有アノマリー |
| `10X_anomaly_categories_*.txt` | 新DLCアノマリー |

> [!TIP]
> Modでは `zz_my_mod_anomalies.txt` のようにzz接頭辞を使うと、バニラファイルの後に読み込まれます。
