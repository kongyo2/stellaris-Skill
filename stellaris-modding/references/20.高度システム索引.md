# 高度システム索引

既存 1-16 は主要な modding 領域をカバーしているが、Stellaris にはそれ以外にも「構造を誤ると壊れやすい」高位システムがある。ここでは、深入り前に確認すべき入口だけを整理する。

## situations

| 項目 | 要点 |
|---|---|
| 主な配置先 | `common/situations/*.txt` |
| 核心 | target、progress、stages、approach、`on_monthly` |
| 重要注意 | 完了/失敗時は `destroy_situation` を忘れない |
| 追加注意 | `target_modifier` は planet target 前提で考える |

読む観点:

- `*_type` / `*_desc` / `*_monthly_change_tooltip` の loc 要件
- `start_value` と `initial_progress` の関係
- `progress_direction`
- `stages` ごとの modifier
- `approach` の `default` と lock/unlock 制御
- `on_monthly` は技術的には専用 on_action であること

## megastructures

| 項目 | 要点 |
|---|---|
| 主な配置先 | `common/megastructures/*.txt` |
| 核心 | upgrade chain、placement、build callbacks |
| 重要注意 | `potential` は country、`possible` は galactic object + `from = country` |
| 追加注意 | `placement_rules`、`build_type`、`starbase` 化の扱い |

読む観点:

- `upgrade_from`
- `build_cost` / `maintenance`
- `on_build_start` / `on_build_complete`
- dismantle 関連 trigger

## zones / zone_slots

| 項目 | 要点 |
|---|---|
| 主な配置先 | `common/zones/*.txt` / `common/zone_slots/*.txt` |
| 核心 | zone_sets、slot 側フィルタ、building_set 連携 |
| 重要注意 | district の `zone_slots` だけでは建築可否は決まらない |
| 追加注意 | `include/exclude` と `included_building_sets/excluded_building_sets` の両方を見る |

読む観点:

- `zone_slot` の `start`
- `included_zone_sets`
- zone の `max_buildings` / `districts_per_building`
- `swap_type` / `swap_type_weight`
- `planet_limit` / `empire_limit`

## solar_system_initializers

| 項目 | 要点 |
|---|---|
| 主な配置先 | `common/solar_system_initializers/*.txt` |
| 核心 | 星系生成、neighbor tree、usage |
| 重要注意 | 記述順がそのまま実行順ではない |
| 追加注意 | `class = random_*` の失敗は実質 `none` 扱いになりうる |

読む観点:

- `usage` / `usage_odds`
- `neighbor_system`
- `mandatory_neighbors`
- 各 `init_effect` の scope

## component_templates

| 項目 | 要点 |
|---|---|
| 主な配置先 | `common/component_templates/*.txt` |
| 核心 | weapon / utility / strike craft / aura / scripted action |
| 重要注意 | `modifier`、`ship_modifier`、`triggered_ship_modifier`、`triggered_ship_design_modifier` の適用先を分ける |
| 追加注意 | 古い restriction 系より `potential` / `possible` を優先する |

読む観点:

- designer 上で見える数値と、実戦用 modifier の違い
- aura の scope
- scripted action の scope と AI 条件

## その他の高価値ディレクトリ

| ドメイン | 主な配置先 | まず確認すること |
|---|---|---|
| policies / edicts | `common/policies` / `common/edicts` | resources、allow、modifier の持ち方 |
| traditions / ascension perks | `common/traditions` / `common/ascension_perks` | unlock と empire-wide modifier の粒度 |
| static modifiers | `common/static_modifiers` | 名前で参照される global/static 効果本体 |
| economic categories | `common/economic_categories` | job/building 出力 modifier の生成元 |

## 読み進め方

1. まずこの索引で対象システムの配置先と注意点を掴む。
2. 次に関連する既存 reference を読む。
3. 共有条件や共有効果が増えそうなら [18.scripted_effects・scripted_triggers・inline_scripts](18.scripted_effects・scripted_triggers・inline_scripts.md) に戻る。
