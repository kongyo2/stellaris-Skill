# Stellaris PDXスクリプト 基礎概念と基本構文

Stellaris PDXスクリプト（Paradoxスクリプト）は、StellarisのMOD制作を強力にサポートするために開発された、独自のドメイン固有言語である。この言語はClausewitzエンジンを基盤とし、Jominiによる機能拡張が加えられている。PDXスクリプトは、宣言的プログラミングと命令的プログラミングのパラダイムを融合させた、特有の階層的なキー・バリュー構造を採用している。

**基本構文の構造:**
```pdx
key = value
object = { nested_content }
array = { value1 value2 value3 }
```

**ファイルエンコーディングと規約:**
スクリプトを正しく動作させるためには、いくつかの規約に従う必要がある。スクリプトファイル（`.txt`など）は**UTF-8**エンコーディングで保存する必要があるが、ローカライゼーションファイル（`.yml`）は**UTF-8 with BOM**で保存しなければならない。また、パーサーはMac OS XおよびLinuxシステムにおいて**大文字と小文字を区別**するため、クロスプラットフォームでの互換性を考慮する際は注意が必要である。コメントは `#` 記号で始まり、公式の（バニラ）コードでは、インデントにはスペースではなく**タブ1つ**を使用するのが標準である。

**重要:** 本ドキュメントを扱うLLMエージェントは、BOM（バイトオーダーマーク）を正確に操作できないため、いかなる場合もBOMの付与を試みてはならない。ローカライゼーションファイル（`.yml`）へのBOM付与（**UTF-8 with BOM**形式での保存）は、全てのコーディング作業が完了した後、必ずユーザー自身が行う。LLMエージェントはこの操作を完全にスキップすること。

**高度な構文機能:**
PDXスクリプトは、より動的で柔軟なスクリプト記述を可能にする高度な機能を備えている。`$PARAMETER$`という構文を用いることで、スクリプトの特定の部分に後から値を埋め込む**パラメータ補間**が可能である。また、`owner.capital_scope.solar_system`のようにドット記法を用いることで、複雑なオブジェクト階層を簡潔に辿る**スコープチェーン**を実現できる。さらに、`@variable_name = value`という形式で、ファイル内でのみ有効なローカル変数を定義することもできる。これらの機能により、スクリプト化されたエフェクトやトリガーを非常に柔軟に構築することが可能になる。
