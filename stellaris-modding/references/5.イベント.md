# イベント (Events)

## 概要

イベントは、Stellarisの物語を紡ぎ、ゲームプレイに動的な変化をもたらすための中核的なスクリプト機能です。特定の条件下で発生し、プレイヤーに情報を提供し、選択肢を提示し、ゲーム世界に様々なエフェクトを及ぼすことで、ゲームの進行を制御します。

すべてのイベントスクリプトは `/events/` ディレクトリ内に `.txt` ファイルとして配置されます。

---

## イベントの基本構造

イベントは `イベントタイプ = { ... }` というブロックで定義されます。以下は `country_event` の例です：

```pdx
# /events/my_mod_events.txt

# ファイルの先頭でnamespaceを宣言する
namespace = my_event

# 国家スコープで実行されるイベント
country_event = {
    # --- 必須パラメータ ---
    id = my_event.1                          # namespace.数値 形式の一意なID
    title = "my_event.1.title"               # イベントウィンドウのタイトル（ローカライゼーションキー）
    desc = "my_event.1.desc"                 # イベントウィンドウの説明文（ローカライゼーションキー）

    # --- 表示関連 ---
    picture = GFX_evt_ship_in_orbit          # 表示される画像
    location = root                          # カメラの注視先となるスコープ
    show_sound = event_ship_bridge           # イベント表示時に再生されるサウンド

    # --- 発生制御 ---
    is_triggered_only = yes                  # on_actionや他のイベントからのみ呼び出し可能
    fire_only_once = no                      # yesにすると一度しか発生しない

    # --- 実行ブロック ---
    trigger = {
        # 条件が満たされない場合、イベントは発生しない
        has_ethic = ethic_materialist
    }

    immediate = {
        # オプション選択を待たずに即座に実行される
        set_country_flag = my_event_started
    }

    # --- 選択肢 （最低一つは必要）---
    option = {
        name = "my_event.1.a"                # 選択肢のテキスト
        
        trigger = {                          # この選択肢が表示される条件
            has_technology = tech_robotics
        }
        
        allow = {                            # 選択可能になる条件（灰色表示）
            NOT = { has_country_flag = robots_outlawed }
        }

        add_resource = { minerals = 100 }    # このオプション選択時のエフェクト
        
        ai_chance = {                        # AIがこの選択肢を選ぶ確率
            factor = 80
            modifier = {
                factor = 1.5
                is_ai_personality = "materialist_enforcers"
            }
        }
    }

    option = {
        name = "my_event.1.b"
        # 何もしない選択肢
    }

    after = {
        # いずれかのオプション選択後に実行されるエフェクト
        clear_country_flag = my_event_started
    }
}
```

## イベントタイプ

イベントは、どのオブジェクトを主体（スコープ）として実行されるかによって多くのタイプに分かれています：

| イベントタイプ | スコープ | 説明 |
|:--|:--|:--|
| `country_event` | 国 | 最も一般的。帝国全体に関わる事象 |
| `planet_event` | 惑星 | 特定の惑星で発生するイベント |
| `ship_event` | 艦船 | 個別の艦船に関連するイベント |
| `fleet_event` | 艦隊 | 艦隊全体に関連するイベント |
| `pop_group_event` | POP | 特定のPOPに関連するイベント (v4.0で`pop_event`から変更) |
| `leader_event` | リーダー | 科学者や提督など特定のリーダー関連 |
| `system_event` | 星系 | 特定の星系で発生するイベント |
| `starbase_event` | 宇宙基地 | 特定の宇宙基地に関連するイベント |
| `pop_faction_event` | 派閥 | 特定の派閥に関連するイベント |
| `situation_event` | 状況 | 特定のSituation内で発生するイベント |
| `first_contact_event` | ファーストコンタクト | ファーストコンタクトプロセス中のイベント |
| `espionage_operation_event` | 諜報作戦 | 諜報作戦内で使用されるイベント |
| `agreement_event` | 協定 | 従属国の協定に関連するイベント |
| `observer_event` | 観測者 | 開発・デバッグ用のイベント |
| `event` | (グローバル) | グローバルスコープのイベント |

## イベントのライフサイクルと実行方法

### 1. イベントの発生（トリガー）

イベントを発生させる方法は主に3つあります。

#### on_action からの呼び出し（推奨）

ゲーム内の特定のアクションにフックしてイベントを呼び出します：

```pdx
# /common/on_actions/my_on_actions.txt
on_game_start = {
    events = {
        my_event.1
    }
}
```

#### 他のイベントやエフェクトからの直接呼び出し

イベントの `option` や `immediate` から別のイベントを呼び出せます：

```pdx
option = {
    name = "Continue the story"
    country_event = { id = my_event.2 }
}
```

遅延付きで呼び出すことも可能です：

```pdx
# 200日から300日後にイベントを発生させる
country_event = { id = my_event.3 days = 200 random = 100 }
```

#### ポーリング（非推奨）

`is_triggered_only = yes` を設定せず `mean_time_to_happen` (MTTH) を使用する方法。パフォーマンスに大きな負荷をかけるため非推奨です。

### 2. 条件の評価

イベントが呼び出されると、以下の順序で条件が評価されます：

1. **`pre_triggers`**: 軽量な条件ブロック。単純な真偽判定のみで構成
2. **`trigger`**: イベント発生のための主要な条件ブロック

### 3. イベントの実行

条件が満たされると、イベントが実行されます：

1. **`immediate`**: イベントウィンドウ表示と同時に即座に実行
2. **`option`**: プレイヤーが選択肢を選ぶ
3. **`after`**: オプション選択後に実行される

## 主要なパラメータ

### 必須・基本パラメータ

| キー | 説明 |
|:--|:--|
| `id` | `namespace.数値` 形式の一意な識別子。文字は使えず先頭の0は無視 |
| `title` | イベントウィンドウのタイトル（ローカライゼーションキー） |
| `desc` | イベントの本文（ローカライゼーションキー）。条件付き切り替え可能 |

### 表示制御

| キー | 説明 |
|:--|:--|
| `picture` | イベントウィンドウに表示する画像 (`interface/*.gfx` で定義) |
| `location` | カメラが注目するスコープ（例: `root`, `from`, `event_target:my_planet`） |
| `hide_window = yes` | イベントウィンドウを表示しない（隠しイベントで必須） |
| `diplomatic = yes` | 外交ウィンドウのような見た目になる |
| `picture_event_data = { ... }` | `diplomatic = yes` 時のポートレート・背景設定 |
| `show_sound` | イベント表示時に再生されるサウンド |

### 発生制御

| キー | デフォルト | 説明 |
|:--|:--:|:--|
| `is_triggered_only` | no | on_actionや他のイベントから明示的に呼び出された場合のみ発生 |
| `fire_only_once` | no | ゲーム全体で一度しか発生しない |
| `abort_trigger = { ... }` | - | 真になるとイベントが強制的に閉じられる |
| `abort_effect = { ... }` | - | `abort_trigger` 発動時に実行されるエフェクト |

> [!IMPORTANT]
> パフォーマンス維持のため、ほぼ全てのイベントで `is_triggered_only = yes` を設定することが推奨されます。

## pre_triggers

`trigger` よりも先に評価される軽量な条件ブロックです。頻繁に呼び出されるイベントでは使用が強く推奨されます。

### 惑星イベント用 pre_triggers

```pdx
planet_event = {
    id = my_event.1
    
    pre_triggers = {
        has_owner = yes              # 惑星に所有者がいるか
        is_homeworld = no            # 母星かどうか
        original_owner = yes         # 元々の所有者かどうか
        is_ai = no                   # 所有者がAIかどうか
        has_ground_combat = no       # 地上戦が行われているか
        is_capital = no              # 首都惑星かどうか
        is_occupied_flag = no        # 占領されているか
    }
    
    # ...
}
```

### その他のスコープ用 pre_triggers

| スコープ | 利用可能な pre_triggers |
|:--|:--|
| **pop** | `common/pop_jobs/000_pretriggers.txt` 参照 |
| **system** | `has_owner`, `is_capital`, `is_occupied_flag` |
| **starbase** | `has_owner`, `is_occupied_flag` |
| **leader** | `has_owner`, `is_idle` |

## オプション (Options)

プレイヤーに提示される選択肢を定義します。

| キー | 説明 |
|:--|:--|
| `name` | 選択肢のテキスト（ローカライゼーションキー） |
| `trigger` | この選択肢が表示されるための条件 |
| `allow` | 選択可能になるための条件（満たさない場合は灰色表示） |
| `ai_chance` | AIがこの選択肢を選ぶ確率。`factor` と `modifier` で構成 |
| `hidden_effect` | ツールチップに表示されないエフェクト |
| `custom_tooltip` | 任意のテキストをツールチップとして表示 |
| `sound` | オプション選択時のサウンド |

### AIの選択確率

```pdx
option = {
    name = "my_event.1.a"
    
    ai_chance = {
        factor = 50                  # 基本値
        modifier = {
            factor = 2               # 条件を満たすと2倍
            has_ethic = ethic_militarist
        }
        modifier = {
            add = 25                 # 加算も可能
            is_at_war = yes
        }
    }
}
```

## 動的な説明文

`desc` はトリガーを使って条件付きで内容を切り替えられます：

```pdx
country_event = {
    id = advisor.1
    title = advisor.1.name
    
    # 条件に応じて異なる説明文を表示
    desc = {
        text = advisor.1.desc_01
        show_sound = generic_democratic_male_intro
        trigger = {
            has_government = gov_representative_democracy
            NOT = { leader = { gender = female } }
        }
    }
    
    desc = {
        text = advisor.1.desc_02
        show_sound = generic_democratic_female_intro
        trigger = {
            has_government = gov_representative_democracy
            leader = { gender = female }
        }
    }
    
    # ...
}
```

## イベントチェーン (Event Chains)

一連の関連するイベントを繋げて、連続した物語を作成する仕組みです。

| キー | 説明 |
|:--|:--|
| `event_chain = <chain_name>` | イベントを特定のイベントチェーンに所属させる |
| `begin_event_chain = { ... }` | `immediate` 内で使用し、新しいイベントチェーンを開始 |

## イベント継承 (Event Inheritance)

v3.4以降、イベントは他のイベントのプロパティを継承できます：

```pdx
# ベースとなるイベント
country_event = {
    id = my_event.base
    title = "Generic Title"
    picture = GFX_evt_default
    option = { name = "OK" }
}

# 継承したイベント
country_event = {
    id = my_event.inherited
    base = my_event.base       # my_event.baseを継承

    desc_clear = yes           # descをクリア
    option_clear = yes         # optionをクリア

    desc = "A new description."
    option = {
        name = "A new option"
        add_resource = { unity = 100 }
    }
}
```

| キー | 説明 |
|:--|:--|
| `base = <event_id>` | 指定したイベントをベースとして読み込む |
| `desc_clear = yes` | ベースイベントの説明をクリア |
| `option_clear = yes` | ベースイベントのオプションをすべてクリア |
| `picture_clear = yes` | ベースイベントの画像をクリア |

## 遅延付きイベント呼び出し

イベントを遅延させて呼び出すには `days` と `random` パラメータを使用します：

```pdx
# 固定の遅延
country_event = { id = my_event.2 days = 30 }

# ランダムな遅延（200〜300日後）
country_event = { id = my_event.3 days = 200 random = 100 }

# scopes指定
country_event = {
    id = my_event.4
    days = 50
    scopes = {
        from = event_target:my_planet
    }
}
```

## 隠しイベント

プレイヤーに見えない裏方処理用のイベント：

```pdx
country_event = {
    id = my_event.hidden
    hide_window = yes
    is_triggered_only = yes
    
    trigger = {
        is_ai = no
        NOT = { has_country_flag = already_processed }
    }
    
    immediate = {
        set_country_flag = already_processed
        # 裏での処理
    }
}
```

> [!TIP]
> 隠しイベントでは `title`, `desc`, `picture`, `option` は不要です。

## フリートアクション

艦隊に対してアクションをキューイングする特殊な機能：

```pdx
fleet_event = {
    id = my_event.fleet
    hide_window = yes
    is_triggered_only = yes
    
    immediate = {
        # 艦隊にアクションをキュー
        queue_actions = {
            move_to = event_target:destination_system
            wait = 50                    # 50日待機
            effect = {
                id = "my_effect_action"
                # エフェクトを実行
            }
        }
    }
}
```

### 利用可能なフリートアクション

| アクション | 説明 |
|:--|:--|
| `orbit_planet = <scope>` | 惑星の軌道に移動 |
| `move_to = <scope>` | 指定スコープに移動 |
| `wait = <days>` | 指定日数待機 |
| `effect = { id = "..." ... }` | エフェクトを実行 |
| `repeat = { ... }` | アクションを繰り返し実行 |
| `find_closest_planet = { ... }` | 最も近い惑星を検索 |

## 変数の使用

イベント内で変数を使用して複雑なロジックを実装できます：

```pdx
country_event = {
    id = my_event.vars
    
    immediate = {
        # 変数を設定
        set_variable = {
            which = my_var
            value = 100
        }
        
        # 変数を操作
        change_variable = {
            which = my_var
            value = trigger:pop_amount    # トリガーの値を使用
        }
        
        # 変数で条件分岐
        if = {
            limit = { check_variable = { which = my_var value >= 150 } }
            add_resource = { minerals = 500 }
        }
    }
}
```

### 変数操作エフェクト

| エフェクト | 説明 |
|:--|:--|
| `set_variable` | 変数を設定 |
| `change_variable` | 加算/減算 |
| `multiply_variable` | 乗算 |
| `divide_variable` | 除算 |
| `modulo_variable` | 剰余 |
| `round_variable` | 最も近い整数に丸め |
| `floor_variable` | 切り捨て |
| `ceiling_variable` | 切り上げ |
| `clear_variable` | 変数をクリア |

## パフォーマンスとベストプラクティス

### 必須事項

- **`is_triggered_only = yes` を常に使用する**: 定期的なポーリングはパフォーマンスに致命的な影響
- **`pre_triggers` を活用する**: 頻繁に呼び出されるイベントでは事前チェックを実施

### ファイル読み込み順序

イベントファイルはファイル名のASCII順に読み込まれますが、同じIDのイベントは最初に読み込まれたものだけが有効になります（First In, Only Served）。

バニラのイベントを上書きしたい場合：

```
!!my_override_events.txt  # ファイル名先頭に ! を付ける
```

### スコープを意識する

イベント内で `root`, `this`, `from`, `prev` やイベントターゲットが何を指しているかを正確に把握することが重要です。

### 構文エラーに注意

括弧 `{}` の対応が取れていないなど、単純な構文エラーは「空のイベントボックスバグ」（タイトルもテキストもないイベントが毎日表示される）の主な原因です。`error.log` を確認しましょう。

> [!CAUTION]
> MTTHを使用したポーリング型イベントは現在ではほとんどのケースで非推奨です。バニラでも遅延付きイベント呼び出しに置き換えられています。

## よく使うトリガー条件

### 国家チェック

```pdx
is_ai = no                           # プレイヤー操作か
has_ethic = ethic_materialist        # 志向チェック
has_country_flag = some_flag         # フラグチェック
has_technology = tech_robotics       # 技術チェック
is_at_war = yes                      # 戦争中か
num_owned_planets >= 5               # 所有惑星数
```

### リーダーチェック

```pdx
leader = { gender = female }
leader = { has_trait = leader_trait_meticulous }
leader = { has_skill_level >= 5 }
```

### 政府・国タイプチェック

```pdx
has_government = gov_representative_democracy
is_country_type = fallen_empire
is_gestalt = yes
is_machine_empire = yes
```

## ファイル命名規則

バニラゲームでは以下の命名パターンが使われています：

| ファイル名パターン | 内容 |
|:--|:--|
| `*_events.txt` | 基本イベント |
| `*_events_1.txt`, `*_events_2.txt` | 連番で分割 |
| `on_action_events_*.txt` | on_action関連 |
| `situation_events.txt` | Situation関連 |

> [!TIP]
> Modでは `zz_my_mod_events.txt` のようにzz接頭辞を使うと、バニラファイルの後に読み込まれます。
