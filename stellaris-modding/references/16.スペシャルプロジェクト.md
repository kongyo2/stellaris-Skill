# スペシャルプロジェクト (Special Projects)

## 概要

スペシャルプロジェクトは、状況ログに表示されプレイヤーが研究・遂行する特別な目標です。アノマリーの調査、考古学サイトの発掘、外交イベントなど、様々な場面で使用されます。

| 要素 | 説明 | 配置場所 |
|:--|:--|:--|
| **スペシャルプロジェクト** | プロジェクトの定義（コスト、要件、完了時の処理） | `/common/special_projects/*.txt` |
| **関連イベント** | 完了・失敗時に発生するイベント | `/events/*.txt` |

---

## 基本構造

```pdx
# /common/special_projects/my_projects.txt

special_project = {
    key = "MY_PROJECT_KEY"           # プロジェクトID（必須）
    
    # === コスト設定（どちらか一方が必要） ===
    cost = 2000                       # 科学ポイントコスト
    cost = {                          # 国スコープで評価される動的コスト
        base = 2000
        modifier = {
            factor = 2
            has_ethic = ethic_militarist
        }
    }
    energy = 100                      # 研究コストとは別の直接コスト
    # または研究コストを 0 にして
    cost = 0
    days_to_research = 180            # 固定日数

    tech_department = society_technology   # 研究コスト型で使う担当部門。days_to_research 型では省略可
    
    # === 表示設定 ===
    picture = GFX_evt_alien_nature    # 画像
    icon = "gfx/interface/icons/situation_log/situation_log_quest.dds"  # アイコン
    sound = no                        # 専用サウンドの有無
    remove_when_completed = yes       # 完了時に自動削除するか

    # === 制御設定 ===
    event_chain = "my_event_chain"    # 関連イベントチェーン
    timelimit = 360                   # 制限日数（-1で無制限）
    location = no                     # 特定の場所が必要か
    project_type = project_type_research    # または project_type_bombardment
    same_option_group_as = { OTHER_PROJECT }  # 排他的選択グループ

    event_scope = ship_event          # ship_event / planet_event / country_event
    desc = {
        text = "MY_PROJECT_KEY_ALT_DESC"
        trigger = { has_ethic = ethic_materialist }
    }

    # === 要件 ===
    requirements = { ... }

    # === コールバック ===
    on_start = { ... }                # 開始時
    on_success = { ... }              # 成功時（必須）
    on_progress_25 = { ... }          # 25%進捗時
    on_progress_50 = { ... }          # 50%進捗時
    on_progress_75 = { ... }          # 75%進捗時
    on_fail = { ... }                 # 失敗時（タイムアウト）
    on_cancel = { ... }               # キャンセル時
    abort_trigger = { ... }           # 中断条件
    fail_trigger = { ... }            # 条件失敗による失敗
    
    # === AI設定 ===
    AI_wait_days = { ... }
}
```

## 主要キーの詳細

### 基本設定

| キー | 型 | 必須 | 説明 |
|:--|:--|:--:|:--|
| `key` | string | ○ | プロジェクトID。イベントやローカライズで使用 |
| `tech_department` | enum | 条件付き | 担当部門。研究コスト型で使う: `physics_technology`, `society_technology`, `engineering_technology` |
| `event_scope` | enum | ○ | スコープ種別: `ship_event`, `planet_event`, `country_event` |

### コスト設定

| キー | 型 | 説明 |
|:--|:--|:--|
| `cost` | int/block | 科学ポイントコスト。固定値または `base/modifier/scaled_modifier` ブロック |
| `days_to_research` | int | 完了までの固定日数。これを使うときは研究コストを `0` にする |
| `<resource> = <int>` | int | `energy` などの直接コスト。研究コストとは別枠 |

> [!IMPORTANT]
> `days_to_research` を使う場合でも、研究コスト欄は `cost = 0` を明示しておくと事故が少ない。動的な研究コストは `cost = { base ... modifier ... }` で組める。

> [!NOTE]
> バニラ documentation では `tech_department` は `cost != 0` のとき relevant とされる。同梱実装には `days_to_research` と併記する例も省略する例もあるため、この skill では「必須」ではなく「研究コスト型で使う指定項目」として扱う。

### 表示設定

| キー | 型 | 説明 |
|:--|:--|:--|
| `picture` | string | 状況ログに表示される画像 (`GFX_` プレフィックス) |
| `icon` | string | 状況ログツリーの左側に表示されるアイコンパス |
| `sound` | bool | 専用サウンドの有無 |

### 制御設定

| キー | 型 | 説明 |
|:--|:--|:--|
| `event_chain` | string | 関連するイベントチェーンID |
| `timelimit` | int | 制限日数。超過で`on_fail`発火。`-1`で無制限 |
| `location` | bool | 特定の場所が必要か |
| `project_type` | enum | `project_type_research` / `project_type_bombardment` |
| `same_option_group_as` | block | 排他的選択グループ。複数のプロジェクトから1つだけ選択可能に |

## requirements（要件）

プロジェクト開始に必要な条件を定義します。通常の条件式（AND/OR）は**使用不可**です。

### 専用コマンド

| コマンド | 説明 | 例 |
|:--|:--|:--|
| `leader` | 必要なリーダー種別 | `leader = scientist` |
| `skill` | リーダーのスキルレベル | `skill > 2` |
| `has_trait` | リーダーの必要特性 | `has_trait = leader_trait_expertise_voidcraft` |
| `shipclass_science_ship` | 科学船の数 | `shipclass_science_ship = 1` |
| `shipclass_constructor` | 建設船の数 | `shipclass_constructor = 1` |
| `shipclass_military` | 軍艦の数 | `shipclass_military = 1` |
| `shipclass_colonizer` | 植民船の数 | `shipclass_colonizer = 1` |
| `shipclass_transport` | 輸送船の数 | `shipclass_transport = 2` |
| `assault_armies` | 強襲軍の数 | `assault_armies = 2` |
| `defense_armies` | 防衛軍の数 | `defense_armies = 1` |
| `fleet_power` | 艦隊戦力 | `fleet_power > 999999` |
| `research_station` | 研究ステーションが必要 | `research_station = yes` |
| `mining_station` | 採掘ステーションが必要 | `mining_station = yes` |
| `observation_station` | 観測所が必要 | `observation_station = yes` |
| `is_founder_species` | 創始者種族のみ | `is_founder_species = yes` |
| `size_requirement` | 特定 ship size の必要数 | `size_requirement = { ship_size = battleship count = 2 }` |
| `has_technology` | 技術要件を追加 | `has_technology = tech_jump_drive_1` |
| `handled_by_event_tooltip` | 条件説明をイベント側 tooltip に委譲 | `handled_by_event_tooltip = SOME_LOC_KEY` |

### 要件の例

```pdx
requirements = {
    shipclass_science_ship = 1
    leader = scientist
    skill > 2
}
```

## コールバック

### スコープ一覧

| コールバック | this | from | fromfrom |
|:--|:--|:--|:--|
| `on_start` | イベントスコープ（船/惑星） | 作成スコープ | - |
| `on_success` | イベントスコープ（船/惑星） | 作成スコープ | - |
| `on_progress_*` | イベントスコープ（船/惑星） | 作成スコープ | - |
| `on_cancel` | 国家（オーナー） | イベントスコープ（存在しない可能性あり） | 作成/位置スコープ |
| `on_fail` | 国家（オーナー） | イベントスコープ（存在しない可能性あり） | 作成/位置スコープ |
| `abort_trigger` | 国家（オーナー） | イベントスコープ（存在しない可能性あり） | 作成/位置スコープ |
| `fail_trigger` | 国家（オーナー） | イベントスコープ（存在しない可能性あり） | 作成/位置スコープ |

### on_success（成功時）

```pdx
on_success = {
    owner = {
        country_event = { id = mymod.1 }
    }
}
```

### on_fail（タイムアウト時）

`timelimit`で指定した日数が経過すると発火します。

```pdx
on_fail = {
    FROM = { planet_event = { id = colony.111 } }
}
```

### abort_trigger（中断条件）

条件がtrueになるとプロジェクトがキャンセルされ、`on_cancel`が発火します。

```pdx
abort_trigger = {
    NOT = { exists = fromfrom }
}
```

## AI_wait_days

AIがプロジェクトを開始するまでの待機日数を設定します。

```pdx
AI_wait_days = {
    weight = 80
    modifier = {
        factor = 0.5
        has_ethic = ethic_fanatic_militarist
    }
}
```

---

## 失敗系コールバックの設計

`on_fail` / `on_cancel` / `abort_trigger` / `fail_trigger` は `this = country` へ寄るため、元の project scope や location scope が揺れやすい。

- 失敗時に元の惑星や対象艦を必ず触りたいなら、**プロジェクト開始時に event target や flag を保存しておく**。
- `from` は存在しないことがある前提で組む。
- 複雑な失敗処理は `on_fail` から hidden event を呼び、そちらで scope を組み直すと安全。

## プロジェクトの制御

### 有効化

```pdx
enable_special_project = {
    name = "MY_PROJECT_KEY"
    location = this          # 場所（通常は惑星やシステム）
    owner = root             # オーナー国家
}
```

### 中断

```pdx
abort_special_project = {
    type = MY_PROJECT_KEY
    location = capital_scope
}
```

### 存在確認

```pdx
has_special_project = MY_PROJECT_KEY
```

## 実践的なサンプル

### 基本的なアノマリー調査プロジェクト

```pdx
special_project = {
    key = "GAS_GIANT_BODIES_PROJECT"
    cost = 0
    days_to_research = 180
    tech_department = society_technology
    picture = GFX_evt_landing_ship

    event_scope = ship_event

    requirements = {
        shipclass_science_ship = 1
        leader = scientist
    }

    on_success = {
        ship_event = { id = anomaly.2501 }
    }

    on_fail = {
    }
}
```

### タイムリミット付きプロジェクト

```pdx
special_project = {
    key = "MUTANT_STALKER_PROJECT"
    cost = 5000
    tech_department = society_technology
    timelimit = 720
    icon = "gfx/interface/icons/situation_log/situation_log_combat.dds"
    picture = GFX_evt_colony_settlement

    event_scope = planet_event

    requirements = {
        assault_armies = 2
    }

    on_success = {
        planet_event = { id = colony.110 }
    }

    on_fail = {
        random_list = {
            50 = {
                FROM = { planet_event = { id = colony.111 } }
            }
            50 = {
                FROM = { planet_event = { id = colony.112 } }
            }
        }
    }
}
```

### 排他的選択グループ

複数のアプローチから1つを選択させる場合：

```pdx
special_project = {
    key = "MIGRATING_FORESTS_1_PROJECT"
    event_chain = "migrating_forests_chain"
    cost = 3500
    tech_department = society_technology
    picture = GFX_evt_alien_nature

    event_scope = planet_event

    on_success = {
        planet_event = { id = colony.2 }
        owner = { end_event_chain = "migrating_forests_chain" }
    }

    on_fail = {
    }
}

special_project = {
    key = "MIGRATING_FORESTS_2_PROJECT"
    event_chain = "migrating_forests_chain"
    cost = 3500
    tech_department = engineering_technology
    picture = GFX_evt_alien_nature

    same_option_group_as = { MIGRATING_FORESTS_1_PROJECT }  # 排他的グループ

    event_scope = planet_event

    on_success = {
        planet_event = { id = colony.3 }
        owner = { end_event_chain = "migrating_forests_chain" }
    }

    on_fail = {
    }
}
```

### 中断条件付きプロジェクト

対象が失われた場合に自動キャンセル：

```pdx
special_project = {
    key = "SHUTTLE_INTERCEPTED"
    cost = 0
    days_to_research = 180
    tech_department = society_technology
    timelimit = 1080
    picture = GFX_evt_big_landing_ship

    event_scope = ship_event

    requirements = {
        shipclass_science_ship = 1
        leader = scientist
    }

    on_success = {
        ship_event = { id = observation.202 }
    }

    on_fail = {
        country_event = { id = observation.203 }
    }

    abort_trigger = {
        exists = event_target:pre_ftl_planet
        event_target:pre_ftl_planet = {
            OR = {
                NOT = { exists = event_target:pre_ftl_country }
                AND = {
                    exists = owner
                    owner = {
                        NOT = { is_same_value = event_target:pre_ftl_country }
                    }
                }
            }
        }
    }
}
```

## ローカライズ

`key`に基づいて以下のキーを定義します：

| キー | 説明 |
|:--|:--|
| `[key]` | プロジェクトタイトル |
| `[key]_DESC` | プロジェクト説明文 |

### 例

```yaml
 MY_PROJECT_KEY:0 "プロジェクトタイトル"
 MY_PROJECT_KEY_DESC:0 "プロジェクトの詳細な説明文"
```
