# 遺物・考古学サイト・アーティファクトアクション

遺物（Relics）、小アーティファクト（Minor Artifacts）、および考古学サイト（Archaeological Sites）は、Stellaris v2.3以降に導入されたメカニクスである。

---

## 遺物（Relics）

**ファイルパス**: `common/relics/xxx.txt`

### データ構造

| プロパティ | 説明 |
|-----------|------|
| `activation_duration` | 発動後の「輝き」持続日数（整数） |
| `portrait` | 遺物アイコンのGFX参照（`interface/relics_view.gfx`で定義） |
| `sound` | 発動時のサウンド参照 |
| `resources` | Economy Unit（リソース産出・発動コスト） |
| `ai_weight` | AIがこの遺物を発動する確率 |
| `triggered_country_modifier` | パッシブ効果の修正子ブロック |
| `score` | 勝利スコア（整数） |
| `active_effect` | 発動時のエフェクトブロック |
| `possible` | 発動可能条件のブロック |
| `can_be_stolen` | 盗まれることが可能か（デフォルト: yes） |

### 例：Ether Drake Trophy

```ruby
@activation_cost = 3000
@triumph_duration = 3600

r_dragon_trophy = {
    activation_duration = @triumph_duration
    portrait = "GFX_relic_dragon_trophy"
    sound = "relic_activation_ether_drake_trophy"

    resources = {
        category = relics
        cost = {
            unity = @activation_cost
        }
    }

    ai_weight = {
        weight = 100
    }

    triggered_country_modifier = {
        potential = {
            always = yes
        }
        country_unity_produces_mult = 0.10
    }

    score = 1000

    active_effect = {
        add_modifier = {
            modifier = "ether_drake_triumph"
            days = @triumph_duration
        }
        custom_tooltip = relic_triumph_cooldown
        hidden_effect = {
            add_modifier = {
                modifier = "relic_activation_cooldown"
                days = @triumph_duration
            }
        }
    }

    possible = {
        custom_tooltip = {
            fail_text = "requires_relic_no_cooldown"
            NOT = { has_modifier = relic_activation_cooldown }
        }
    }
}
```

> [!IMPORTANT]
> 全ての遺物は発動時に `relic_activation_cooldown` 修正子を付与し、`possible` ブロックでこの修正子がある場合は発動不可とすること。これにより遺物UIと「発動可能」アラートメッセージが正常に動作する。

### inline_scriptを使用した例

```ruby
possible = {
    inline_script = {
        script = relics/activation_checks
        RELIC = r_the_modularium
    }
}
```

## アーティファクトアクション（Artifact Actions）

**ファイルパス**: `common/artifact_actions/xxx.txt`

Ancient Relics DLCがない場合、アーティファクトアクションはUIに表示されず使用不可。

### データ構造

| プロパティ | 説明 |
|-----------|------|
| `resources` | Economy Unit（アクションコスト、通常は小アーティファクト） |
| `ai_weight` | AIがこのアクションを使用する確率 |
| `potential` | UIに表示するかどうかの条件 |
| `allow` | アクション実行可能の条件 |
| `effect` | 実行時のエフェクトブロック |

### 例：Arcane Deciphering

```ruby
@minor_artifact_decision_cost = 100
@minor_artifact_decision_duration = 3600

artifact_arcane_deciphering = {
    resources = {
        category = artifact_actions
        cost = {
            minor_artifacts = 500
        }
    }

    ai_weight = {
        weight = 100
        modifier = {
            factor = 1.5
            has_ethic = ethic_materialist
        }
    }

    potential = {
        has_technology = tech_arcane_deciphering
    }

    allow = {
        custom_tooltip = {
            fail_text = "requires_arcane_deciphering_no_cooldown"
            NOT = { has_modifier = arcane_deciphering_cooldown }
        }
    }

    effect = {
        custom_tooltip = artifact_arcane_deciphering_effect
        hidden_effect = {
            add_modifier = {
                modifier = "arcane_deciphering_cooldown"
                days = 720
            }
            owner = {
                country_event = { id = ancrel.10000 }
            }
        }
    }
}
```

### 特殊なアーティファクトアクション

- **ロック状態アクション**: 技術「Arcane Deciphering」取得前は「???」として表示。これらは別個のアクションで、取得前のみ表示され発動不可。
- **惑星経由アクション**: 惑星ディシジョン経由でのみ実行可能なアクション。発動不可として設定され、実際の効果はディシジョンで実装。

## 考古学サイト（Archaeological Sites）

**ファイルパス**: `common/archaeological_site_types/xxx.txt`

考古学サイトの完全な実装は、イベントModdingおよびダイナミックModdingに大きく依存する。

### データ構造

| プロパティ | 説明 |
|-----------|------|
| `desc` | サイト説明のローカライズキー（scope = site） |
| `picture` | サイト画像のGFX参照 |
| `max_instances` | 銀河内の最大インスタンス数（`create_archaeological_site = random`使用時のみチェック） |
| `weight` | ランダム重み（`create_archaeological_site = random`使用時） |
| `stages` | サイトのステージ総数（`stage`ブロック数と一致必須） |
| `potential` | 発掘可能かの条件（scope = fleet, from = site） |
| `allow` | 船が発掘に従事できるかの条件（scope = fleet, from = site） |
| `visible` | 帝国に表示するかの条件（scope = country, from = site） |
| `stage` | ステージ定義（順序依存）|
| `on_roll_failed` | ロール失敗時のエフェクト（scope = fleet, from = site） |
| `on_create` | サイト作成時のエフェクト（scope = site） |
| `on_visible` | サイト表示時のエフェクト（scope = country, from = site） |

### stageブロック

| プロパティ | 説明 |
|-----------|------|
| `difficulty` | 難易度（整数または `{ min = <int> max = <int> }`） |
| `icon` | ステージアイコンのGFX参照（rune icon gfx type） |
| `event` | ステージクリア時に発火するfleet_event ID |

### 例：Zroni Digsite 1

```ruby
zroni_digsite_1 = {
    desc = zroni_digsite_1_desc
    picture = GFX_evt_overgrown_city
    stages = 3
    max_instances = 1

    allow = {
        is_ship_class = shipclass_science_ship
        exists = leader
    }

    visible = {
        has_country_flag = zroni_intro
        default_site_visible_trigger = yes
    }

    stage = {
        difficulty = 1
        icon = GFX_archaeology_runes_F1
        event = ancrel.2
    }
    stage = {
        difficulty = 1
        icon = GFX_archaeology_runes_F2
        event = ancrel.3
    }
    stage = {
        difficulty = 2
        icon = GFX_archaeology_runes_F3
        event = ancrel.4
    }

    on_roll_failed = {
        from = {
            standard_archaeological_site_on_roll_failed = { RANDOM_EVENTS = all_random_events }
        }
    }
}
```

> [!NOTE]
> `standard_archaeological_site_on_roll_failed` はscripted effectである。

### 難易度形式

```ruby
# 固定値
difficulty = 3

# ランダム範囲
difficulty = { min = 1 max = 4 }
```

### weight形式

```ruby
# 固定値
weight = 0

# 条件付き
weight = {
    base = 0
    modifier = {
        add = 50
        is_planet_class = pc_tropical
        has_moon = yes
    }
}
```

## よく使われるトリガー・エフェクト

### サイト関連トリガー

```ruby
# サイトが存在するか
has_archaeological_site = <サイトID>

# サイトの現在ステージ
archaeological_site_stage = <整数>

# サイトのクリア済みステージ
archaeological_site_stages_cleared = <整数>
```

### サイト関連エフェクト

```ruby
# サイト作成
create_archaeological_site = <サイトID | random>

# サイト破壊
destroy_archaeological_site = yes

# サイトステージ進行
set_archaeological_site_stage = <整数>
```

### 遺物関連

```ruby
# 遺物付与
add_relic = <遺物ID>

# 遺物所持チェック
has_relic = <遺物ID>

# 遺物発動
activate_relic = <遺物ID>
```
