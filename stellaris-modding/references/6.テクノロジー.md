# テクノロジー (Technology)

## 概要

テクノロジーは、Stellarisにおける帝国の進化と発展を定義する中核的なシステムです。帝国は研究を通じて新しい技術を獲得し、新しい建造物、艦船コンポーネント、惑星ディシジョンなどをアンロックできます。

技術データは `/common/technology/` ディレクトリ内のファイルで管理されます。

- `tier/00_tier.txt`: 技術ティアの定義
- `category/00_category.txt`: 技術カテゴリの定義
- `*_tech.txt`: 具体的な技術定義

---

## 技術の基本構造

```pdx
# /common/technology/my_mod_tech.txt

tech_my_custom_technology = {
    # --- 必須パラメータ ---
    cost = @tier2cost1                       # 研究コスト
    area = physics                           # 研究エリア (physics/society/engineering)
    tier = 2                                 # ティア (0-5)
    category = { computing }                 # カテゴリ
    weight = @tier2weight1                   # 出現ウェイト

    # --- 前提・条件 ---
    prerequisites = { "tech_basic_science_lab_2" }  # 前提技術
    potential = {                            # 出現条件
        is_gestalt = no
    }

    # --- 研究完了時の効果 ---
    modifier = {
        planet_jobs_physics_research_produces_mult = 0.10
    }

    # --- 出現確率調整 ---
    weight_modifier = {
        modifier = {
            factor = 1.5
            research_leader = {
                area = physics
                has_trait = leader_trait_expertise_computing
            }
        }
    }

    # --- AI関連 ---
    ai_weight = {
        factor = 1.0
    }
    ai_update_type = all                     # 研究完了時にAIの設計を更新

    # --- オプションのタグ ---
    is_rare = no                             # レア技術（紫枠）
    is_dangerous = no                        # 危険技術（赤枠）
    is_reverse_engineerable = yes            # デブリから取得可能
}
```

---

## 主要パラメータ詳細

### 必須パラメータ

| キー | 説明 |
|:--|:--|
| `cost` | 研究に必要なポイント。固定値または `@tier1cost1` などの変数 |
| `area` | `physics`（物理学）、`society`（社会学）、`engineering`（工学）のいずれか |
| `tier` | 技術階層 (0-5)。前のティアを一定数研究すると次のティアが解放 |
| `category` | 技術カテゴリ。科学者の専門分野と一致すると研究速度ボーナス |
| `weight` | 研究選択肢に出現する確率の重み |

### ティアとコスト

`00_tier.txt` で定義されており、`previously_unlocked` で解放条件が設定されています：

```pdx
# /common/technology/tier/00_tier.txt
2 = {
    previously_unlocked = 6    # ティア1の技術を6つ研究すると解放
    weight_modifier = {
        base = 1
    }
}
```

| ティア | 目安コスト | 目安ウェイト |
|:--:|:--:|:--:|
| 0 | 開始技術 | - |
| 1 | 2000-3000 | 90-100 |
| 2 | 4000-6000 | 70-80 |
| 3 | 8000-12000 | 50-60 |
| 4 | 16000-24000 | 30-40 |
| 5 | 32000-48000 | 20-30 |

### 条件付きコスト

```pdx
cost = {
    base = 4000
    modifier = {
        factor = 0.75              # 25%割引
        is_active_resolution = "resolution_research_subsidies"
    }
}
```

## カテゴリ

`00_category.txt` で定義されています。新規カテゴリの追加は不可（ハードコード）。

### 既存カテゴリ一覧

| カテゴリ | アイコン | 主な技術 |
|:--|:--|:--|
| `materials` | 素材 | 合金、建造物素材 |
| `propulsion` | 推進 | FTL、エンジン |
| `voidcraft` | 艦艇工学 | 艦船、宇宙基地 |
| `industry` | 産業 | 採掘、生産効率 |
| `field_manipulation` | フィールド操作 | シールド、エネルギー |
| `particles` | 粒子物理 | レーザー、ゲートウェイ |
| `computing` | 計算機工学 | AI、自動化 |
| `psionics` | 超能力 | サイオニック技術 |
| `new_worlds` | 新世界 | テラフォーミング、居住 |
| `statecraft` | 政治学 | 行政、外交 |
| `biology` | 生物学 | 遺伝子改変、成長 |
| `military_theory` | 軍事理論 | 戦術、軍事施設 |
| `archaeostudies` | 考古学 | 遺物解析（Ancient Relics DLC） |

### カテゴリ研究速度modifier

技術カテゴリごとに、研究速度を触るためのmodifier名が自動で対応付けられている。たとえば `computing` なら `category_computing_research_speed_mult`、`biology` なら `category_biology_research_speed_mult` を使う。

国是、評議員、静的modifier、イベント効果からカテゴリ研究速度を操作したいときは、この命名に合わせる。

## 技術タグ

| タグ | デフォルト | 説明 |
|:--|:--:|:--|
| `start_tech = yes` | no | 帝国の開始技術として設定 |
| `is_rare = yes` | no | 紫枠のレア技術（出現率自体には影響しない） |
| `is_dangerous = yes` | no | 赤枠の危険技術（危機イベントのトリガーに使用） |
| `is_reverse_engineerable = yes` | yes | デブリ解析で獲得可能 |
| `is_insight = yes` | no | 洞察（Insight）技術として扱う |

> [!NOTE]
> `is_rare` 自体は出現確率に影響しません。「技術的優位」アセンションパークなど、レア技術を参照する効果の対象になります。

## potential ブロック

技術が研究選択肢に現れるための**絶対条件**を定義します：

```pdx
potential = {
    # DLC所持チェック
    host_has_dlc = "Utopia"
    
    # 帝国タイプチェック
    is_gestalt = no
    is_wilderness_empire = no
    
    # その他の条件
    has_origin = origin_void_dwellers
}
```

> [!IMPORTANT]
> `potential` 内の条件が一つでも満たされない場合、その技術は絶対に選択肢に現れません。

## weight_modifier ブロック

出現確率を動的に調整します：

```pdx
weight_modifier = {
    # 特定志向で出現率上昇
    modifier = {
        factor = 2.0
        has_ethic = ethic_materialist
    }
    
    # 反対の志向で出現率低下
    modifier = {
        factor = 0.5
        has_ethic = ethic_spiritualist
    }
    
    # 科学者の特性による調整
    modifier = {
        factor = 1.5
        research_leader = {
            area = physics
            has_trait = leader_trait_expertise_computing
        }
    }
    
    # 経過年数による調整
    modifier = {
        factor = 0.1
        NOT = { years_passed > 50 }
    }
    
    # レア技術の共通処理
    inline_script = {
        script = technologies/rare_technologies_weight_modifiers
        TECHNOLOGY = tech_my_technology
    }
}
```

### 乗算と加算

| 演算子 | 例 | 説明 |
|:--|:--|:--|
| `factor` | `factor = 2` | ウェイトを2倍（+100%） |
| `factor` | `factor = 0.5` | ウェイトを半減（-50%） |
| `add` | `add = 20` | ウェイトに20を加算 |

## prerequisites（前提技術）

```pdx
# 単純な前提
prerequisites = { "tech_basic_science_lab_2" }

# 複数の前提（すべて必要）
prerequisites = { "tech_gateway_activation" "tech_mega_engineering" }

# OR条件（いずれか一つ）
prerequisites = {
    "tech_foundational_theory"
    OR = {
        "tech_droids"
        "tech_psionic_theory"
    }
}
```

## ai_update_type

研究完了時にAIの艦船設計を更新するかどうか：

| 値 | 説明 |
|:--|:--|
| `military` | 軍艦と宇宙基地の設計を更新 |
| `all` | 軍事 + 民間船（科学船、建設船など）も更新 |
| (未指定) | コンポーネント技術でも自動更新しない |

```pdx
tech_shields_4 = {
    # ...
    ai_update_type = all    # AIはこの技術研究後に全艦船設計を更新
}
```

## ai_weight ブロック

AIの研究優先度を設定：

```pdx
ai_weight = {
    factor = 1.0
    
    # 軍国主義AIは優先
    modifier = {
        factor = 2.0
        is_militarist = yes
    }
    
    # 平和主義AIは後回し
    modifier = {
        factor = 0.5
        is_pacifist = yes
    }
    
    # 好みの武器タイプと合致する場合
    modifier = {
        factor = @favored_weapon_ai_tech_factor
        is_preferred_weapons = weapon_type_energy
    }
}
```

## 繰り返し技術 (Repeatable Technologies)

無限に研究可能な技術を定義：

```pdx
tech_repeatable_improved_shield_output = {
    area = physics
    cost = @repeatableTechBaseCost
    cost_per_level = @repeatableTechLevelCost    # レベルごとの追加コスト
    tier = @repeatableTechTier
    category = { field_manipulation }
    ai_update_type = all
    levels = -1                                   # -1 = 無限に研究可能
    prerequisites = { "tech_shields_4" }
    weight = @repeatableTechWeight

    # 同グループの技術が同時に選択肢に出にくくする
    weight_groups = { repeatable }
    mod_weight_if_group_picked = {
        repeatable = 0.01
    }

    modifier = {
        ship_shield_mult = 0.05                   # レベル毎に+5%
    }
}
```

### 有限回数の繰り返し技術

```pdx
tech_repeatable_reduced_building_cost = {
    # ...
    levels = 5                                    # 最大5回まで研究可能
    cost_per_level = @repeatableTechLevelCost
    # ...
}
```

> [!TIP]
> `levels > 1` の場合、`cost_per_level` の定義が必須です。

## technology_swap

特定条件下で技術を別の技術に置き換え：

```pdx
tech_corvettes = {
    # 通常の定義...
    
    technology_swap = {
        name = tech_bioship_corvettes            # 置き換え後の技術ID
        inherit_icon = yes                       # アイコンを継承
        inherit_effects = no                     # 効果は継承しない
        trigger = {
            country_uses_bio_ships = yes         # バイオ艦船使用時
        }
        # 置き換え後の独自設定
        modifier = {
            ship_hull_mult = 0.10
        }
        weight = {
            factor = 1
        }
    }
}
```

> `technology_swap` で `area` や `category` を指定した場合、それらは元技術への追記ではなく置き換えとして扱う。特に `category = { ... }` は元カテゴリを全置換するため、残したいカテゴリがあるなら明示的に再記述する。

## feature_flags

ハードコードされたゲーム機能をアンロック（MODで新規追加不可）：

```pdx
tech_gateway_activation = {
    # ...
    feature_flags = { gateway_activation }       # ゲートウェイ起動を可能に
}

tech_mega_engineering = {
    # ...
    feature_flags = { megaengineering }          # 巨大構造物建造を可能に
}
```

## prereqfor_desc

この技術が何の前提となるかを表示：

```pdx
tech_cruisers = {
    # ...
    prereqfor_desc = {
        ship = {
            title = "CRUISER_PREREQ_TITLE"       # ローカライゼーションキー
            desc = "CRUISER_PREREQ_DESC"
        }
    }
}

tech_archaeo_detection_scrambler = {
    # ...
    prereqfor_desc = {
        hide_prereq_for_desc = component         # 通常のコンポーネント表示を隠す
        custom = {
            title = "TECH_UNLOCK_SCRAMBLER_TITLE"
            desc = "TECH_UNLOCK_SCRAMBLER_DESC"
        }
    }
}
```

## weight_groups

同グループの技術が研究選択肢に同時出現しにくくする：

```pdx
# 技術A
tech_laser_upgrade_1 = {
    weight_groups = { energy_weapons }
    mod_weight_if_group_picked = {
        energy_weapons = 0.1                     # 同グループが既にあれば0.1倍
    }
}

# 技術B
tech_laser_upgrade_2 = {
    weight_groups = { energy_weapons }
    mod_weight_if_group_picked = {
        energy_weapons = 0.1
    }
}
```

## modifier ブロック

研究完了時に適用される永続的効果：

```pdx
modifier = {
    # ジョブ産出ボーナス
    planet_jobs_energy_produces_mult = 0.05
    planet_jobs_physics_research_produces_mult = 0.10
    
    # 艦船性能
    ship_shield_mult = 0.05
    ship_hull_mult = 0.10
    
    # 武器性能
    weapon_type_energy_weapon_damage_mult = 0.05
    
    # カスタムツールチップ
    custom_tooltip = "my_tech_modifier_tooltip"
}
```

### modifier ブロックの高度設定

`modifier` では通常のmodifier値だけでなく、表示制御用の補助キーも使える。

| キー | 説明 |
|:--|:--|
| `description` | 効果説明のローカライズキー |
| `description_parameters = { POINTS = <int> }` | 説明文へ値を差し込む |
| `custom_tooltip` | 自前の補足ツールチップ |
| `show_only_custom_tooltip = yes` | 自動生成のmodifier一覧を隠し、カスタム説明だけ見せる |

数値modifierよりも「何を解禁するか」を説明したい技術では、`show_only_custom_tooltip = yes` とローカライズを組み合わせるとツールチップを崩しにくい。

## 実践的なサンプル

### 基本的な技術

```pdx
tech_advanced_cloning = {
    cost = 5000
    area = society
    tier = 2
    category = { biology }
    prerequisites = { "tech_genome_mapping" }
    weight = 80
    
    potential = {
        NOT = { has_ethic = ethic_fanatic_spiritualist }
    }

    modifier = {
        pop_growth_speed = 0.10
    }

    weight_modifier = {
        modifier = {
            factor = 1.5
            research_leader = {
                area = society
                has_trait = leader_trait_expertise_biology
            }
        }
        modifier = {
            factor = 0.5
            years_passed < 50
        }
    }

    ai_weight = {
        factor = 1.2
        modifier = {
            factor = 2
            has_ethic = ethic_xenophile
        }
    }

    is_rare = yes
}
```

### 開始技術

```pdx
tech_basic_science_lab_1 = {
    cost = @tier0cost1
    area = physics
    tier = 0
    category = { computing }
    start_tech = yes                             # 開始技術フラグ

    weight_modifier = {
        factor = 1000
    }

    ai_weight = {
        weight = 10000
    }

    # 特定起源では開始技術にしない
    starting_potential = {
        is_low_tech_start = no
    }
}
```

### DLC限定技術

```pdx
tech_dyson_sphere = {
    area = physics
    cost = @tier5cost3
    tier = 5
    category = { field_manipulation }
    ai_update_type = all
    prerequisites = { "tech_mega_engineering" }
    weight = @tier5weight3
    is_rare = yes

    potential = {
        host_has_dlc = "Utopia"                  # Utopia DLC必須
    }

    weight_modifier = {
        factor = 0                               # 通常は選択肢に出ない
        # アセンションパークでアンロック
    }

    ai_weight = {
        factor = @best_megastructure_ai_tech_factor
    }
}
```

### 条件による出現調整

```pdx
tech_gateway_activation = {
    cost = @tier4cost3
    area = physics
    tier = 4
    category = { particles }
    is_rare = yes
    is_reverse_engineerable = no
    weight = @tier4weight3
    prerequisites = { "tech_hyper_drive_3" }

    feature_flags = { gateway_activation }

    weight_modifier = {
        factor = 0.5
        
        # ゲートウェイを見たことがない＆誰も持っていない場合は出現しない
        modifier = {
            factor = 0
            NOR = {
                OR = {
                    has_seen_any_bypass = gateway
                    has_seen_any_bypass = lgate
                }
                any_relation = {
                    is_country_type = default
                    has_communications = ROOT
                    has_technology = tech_gateway_activation
                }
            }
        }
        
        # ゲートウェイを所有していれば出現率上昇
        modifier = {
            factor = 4
            OR = {
                owns_any_bypass = gateway
                owns_any_bypass = lgate
            }
        }
        
        inline_script = {
            script = technologies/rare_technologies_weight_modifiers
            TECHNOLOGY = tech_gateway_activation
        }
    }

    ai_weight = {
        modifier = {
            factor = 2
            is_ai = no                           # プレイヤー優先
        }
    }
}
```

## ベストプラクティス

### ID命名規則

- 接頭辞を付けて一意性を確保: `my_mod_tech_advanced_cloning`
- バニラ命名パターン: `tech_[エリア]_[名前]_[段階]`

### パフォーマンス

- `potential` でDLCチェックや帝国タイプを制限し、不要な評価を減らす
- `weight_modifier` は複雑になりすぎないようにする

### 互換性

- 既存技術の上書きは避け、新規技術として追加する
- 既存技術を前提にする場合は `potential` でその技術の存在を確認

### ファイル命名

| パターン | 内容 |
|:--|:--|
| `00_eng_tech.txt` | 工学技術 |
| `00_phys_tech.txt` | 物理学技術 |
| `00_soc_tech.txt` | 社会学技術 |
| `00_*_tech_repeatable.txt` | 繰り返し技術 |
| `00_*_weapon_tech.txt` | 武器技術 |

> [!TIP]
> MODファイルは `zz_my_mod_tech.txt` のようにzz接頭辞を使うと、バニラファイルの後に読み込まれます。

## よく使うトリガー・条件

### 帝国チェック

```pdx
is_gestalt = yes                     # 集合意識/機械知性
is_machine_empire = yes              # 機械知性
is_hive_empire = yes                 # 集合意識
is_wilderness_empire = yes           # 野生の帝国
has_ethic = ethic_materialist        # 志向チェック
is_militarist = yes                  # 軍国主義系志向
```

### DLCチェック

```pdx
host_has_dlc = "Utopia"
host_has_dlc = "Megacorp"
has_ancrel = yes                     # Ancient Relics
has_overlord_dlc = yes               # Overlord
```

### 科学者チェック

```pdx
research_leader = {
    area = physics
    has_trait = leader_trait_expertise_computing
    has_level >= 4
}

has_trait_in_council = { TRAIT = leader_trait_spark_of_genius }
```

### 技術・施設チェック

```pdx
has_technology = tech_mega_engineering
num_buildings = {
    type = building_research_lab_1
    value >= 3
}
```
