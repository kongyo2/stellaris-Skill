# 経済カテゴリ・triggered_modifier・zones

jobs、buildings、districts、zones の調整では、個別定義より上位の抽象化が支配的になる。とくに `economic_categories` と `triggered_modifier` の継承を読み違えると、見た目どおりに効果が出ない。

## 経済カテゴリ

経済カテゴリは、job・building・planet・ship・megastructure などの resource table を束ねる。

### 何に使うか

- modifier を自動生成する。
- AI の予算計画に使う。

### 重要ルール

- 子カテゴリは親カテゴリの `_mult` を継承する。
- 子カテゴリは親カテゴリの `_add` を継承しない。
- 特定 job だけを強化したいなら、その job 専用の子カテゴリを切る。
- `modifier_category` は、使う最下層の scope に寄せる。
- `modifier_category` を使うなら、triggered modifier より前に宣言する。

### 実務上の意味

- `planet_miners_*` を触ると、同カテゴリの job 全体へ波及する。
- 1 job だけ別扱いにしたいなら job 側の `category` 再設計が必要。
- jobs 出力調整は `resources` だけでなく、カテゴリ継承も見る。

### 経済カテゴリ側の triggered modifier

経済カテゴリは `triggered_produces_modifier` / `triggered_cost_modifier` / `triggered_upkeep_modifier` を生成できます。

- 条件付きで category 由来 modifier を生やしたいときに使う。
- resource table 計算時に走るため、trigger は軽く保つ。
- 生成された loc key は `mod_...` で始まる。必要なら同名 icon を `gfx/interface/icons/modifiers` に置く。

## triggered_modifier

triggered modifier は、通常 modifier と同じ見た目でも「条件を満たしたときだけ適用される」。

### 典型構造

- `potential`
- `modifier`
- `mult` または `multiplier`

### 実務ルール

- ツールチップに全てが自然表示されるとは限らない。
- プレイヤーに見せる必要がある条件差分は `custom_tooltip` で補う。
- 複雑な trigger を大量にぶら下げると、読みやすさと性能が悪化する。
- `triggered_*_modifier` は direct modifier keys でも `modifier = {}` ラッパーでも書ける。バニラは direct 形式を多用する。

### 使いどころ

- 帝国タイプ別の jobs 差分
- 特定惑星状態でだけ追加される amenities / upkeep / production
- leader trait や resolution の条件付き説明

## jobs / buildings / districts の接点

- jobs は `possible_pre_triggers` と `possible` を分けて考える。
- jobs は `swappable_data` で名称・アイコン・説明を切り替えられる。
- building / district は `triggered_planet_modifier` や `triggered_country_modifier` で empire type 差分を吸収しやすい。

## 4.0 zones

Phoenix 4.0 系では、district と building の間に zone の層がある。

### district 側

- `default_starting_district`
- `zone_slots`
- `convert_to`

### zone 側

- `include` / `exclude`
- `included_building_sets` / `excluded_building_sets`
- `zone_sets`
- `max_buildings`
- `districts_per_building`
- `planet_limit` / `empire_limit`
- `swap_type` / `swap_type_weight`
- `district_planet_modifier`
- `triggered_district_planet_modifier`
- `district_country_modifier`

### building 側

- `building_sets`

## 設計指針

- 個別 building を増やす前に、zone / building_set で吸収できないか確認する。
- 出力倍率を盛る前に、その job の経済カテゴリがどこまで広がるか確認する。
- triggered modifier で hidden な差分を増やすほど、補助 tooltip が必要になる。
- zone が見えない/建てられないときは、district の `zone_slots` だけでなく `zone_slot` 側の `included_zone_sets` と zone 側の `include/exclude` も確認する。
