# 建造物とディストリクト (Buildings and Districts)

## 概要

建造物 (Buildings) とディストリクト (Districts) は、惑星の経済と機能を決定する重要な要素です。建造物は惑星ビュー内の建造物スロットに配置され、ディストリクトは惑星のサイズに依存するスタック可能なタイルとして配置されます。

| カテゴリ | パス |
|:--|:--|
| 建造物 | `common/buildings/*.txt` |
| ディストリクト | `common/districts/*.txt` |
| アイコン | `gfx/interface/icons/buildings/*.dds` |
| ローカライズ | `localisation/*_l_<LANG>.yml` |

## 建造物の基本構造

```pdx
# /common/buildings/my_buildings.txt

building_sample = {
    # --- 基本設定 ---
    base_buildtime = 360                        # 建設日数（及びアップグレード日数）
    
    # --- アイコン ---
    icon = building_sample                      # gfx/interface/icons/buildings/内のファイル名
    
    # --- カテゴリ ---
    category = manufacturing                    # 建造物カテゴリ（フィルタ用）
    
    # --- 建造物セット（4.0+） ---
    building_sets = {
        manufacturing
    }
    
    # --- 首都関連 ---
    capital = no                                # 首都建造物か
    capital_tier = 1                           # 首都ティア（1-4）
    
    # --- 建設・解体制限 ---
    can_build = yes                             # 建造可能か
    can_demolish = yes                          # 解体可能か
    can_be_ruined = yes                         # 廃墟になりうるか
    can_be_disabled = yes                       # 無効化可能か
    
    # --- 配置優先度 ---
    position_priority = 200                     # スロット配置順（小さいほど前）
    
    # --- 建造数制限 ---
    base_cap_amount = 1                         # 惑星あたりの上限
    empire_limit = {                            # 帝国全体での上限
        base = 1
        modifier = {
            add = 1
            has_technology = tech_mega_engineering
        }
    }
    is_capped_by_modifier = yes                 # モディファイアで上限追加可能
    
    # --- FTL阻害装置 ---
    planetary_ftl_inhibitor = yes               # FTL阻害装置として機能
    
    # --- 技術要件 ---
    prerequisites = {
        "tech_alloys"
    }
    show_tech_unlock_if = {                     # 技術ツールチップに表示する条件
        is_regular_empire = yes
    }
    
    # --- アップグレード ---
    upgrades = {
        building_sample_2
    }
    
    # --- 条件 ---
    potential = {                               # 表示条件（惑星スコープ）
        exists = owner
        owner = { is_regular_empire = yes }
    }
    allow = {                                   # 建設可能条件（惑星スコープ）
        num_pops >= 10
    }
    
    # --- トリガー ---
    abort_trigger = { }                         # 建設キューから除外する条件
    ruined_trigger = { }                        # 廃墟になる条件
    destroy_trigger = { }                       # 破壊される条件
    
    # --- 変換先 ---
    convert_to = {
        building_hive_sample                    # 破壊時の代替建造物
    }
    
    # --- モディファイア ---
    planet_modifier = {                         # 惑星に適用する修正値
        job_artisan_add = 2
    }
    country_modifier = {                        # 帝国に適用する修正値
        country_resource_max_add = 5000
    }
    
    # --- 条件付きモディファイア ---
    triggered_planet_modifier = {
        potential = {
            exists = owner
            owner = { is_gestalt = yes }
        }
        modifier = {
            job_artisan_drone_add = 2
        }
    }
    triggered_country_modifier = {
        potential = {
            exists = owner
            owner = { has_technology = tech_mega_engineering }
        }
        country_resource_max_add = 10000
    }
    
    # --- リソース ---
    resources = {
        category = planet_buildings
        cost = {
            minerals = 500
            alloys = 200
        }
        upkeep = {
            energy = 5
        }
        produces = {
            alloys = 10
        }
        # 条件付きリソース
        upkeep = {
            trigger = {
                has_modifier = forge_world
            }
            energy = 10
            minerals = 5
        }
    }
    
    # --- ツールチップ ---
    triggered_desc = {
        trigger = {
            exists = owner
            owner = { is_regular_empire = yes }
        }
        text = "job_artisan_effect_desc"
    }
    
    # --- エフェクト ---
    on_queued = { }                             # キューに追加時
    on_unqueued = { }                           # キューから削除時
    on_built = { }                              # 建設完了時
    on_destroy = { }                            # 破壊時
    
    # --- AI設定 ---
    ai_weight = {
        weight = 100
        modifier = {
            factor = 2
            has_designation = col_industrial
        }
    }
    ai_resource_production = {
        alloys = 10
        trigger = {
            always = yes
        }
    }
    additional_ai_weight = 100                  # 追加AI重み
}
```

## 建造物カテゴリ

| カテゴリ | 説明 |
|:--|:--|
| `pop_assembly` | POP組立関連（ロボット工場など） |
| `government` | 政府建造物（首都など） |
| `resource` | 資源生産（採掘ネットワークなど） |
| `manufacturing` | 製造関連（合金鋳造所など） |
| `research` | 研究施設 |
| `trade` | 交易関連 |
| `amenity` | 快適度関連（ホロシアターなど） |
| `unity` | 統合力関連（記念碑など） |
| `army` | 軍事施設（要塞など） |

## ディストリクトの基本構造

```pdx
# /common/districts/my_districts.txt

district_sample = {
    base_buildtime = 480
    
    # --- アイコン ---
    icon = district_sample
    
    # --- 表示条件 ---
    potential = {                               # 表示条件（所有者あり）
        uses_district_set = standard
    }
    show_on_uncolonized = {                    # 未植民惑星での表示条件
        uses_district_set = standard
    }
    
    # --- 建設条件 ---
    allow = {
        NOT = { has_modifier = resort_colony }
    }
    
    # --- 変換設定（4.0+） ---
    convert_to = {
        district_hive_sample
    }
    conversion_ratio = 0.5                      # 変換比率（端数切り捨て）
    
    # --- 生成数制限 ---
    min_for_deposits_on_planet = 1              # 惑星生成時の最小数
    max_for_deposits_on_planet = 15             # 惑星生成時の最大数
    is_capped_by_modifier = yes
    
    # --- 4.0ゾーン設定 ---
    default_starting_district = yes             # このセットのプライマリディストリクト
    zone_slots = {                              # ゾーンスロット
        slot_city_government
        slot_city_01
        slot_city_02
    }
    
    # --- モディファイア ---
    planet_modifier = {
        planet_housing_add = 5
        job_clerk_add = 2
    }
    
    triggered_planet_modifier = {
        potential = {
            exists = owner
            owner = { is_hive_empire = yes }
        }
        modifier = {
            job_maintenance_drone_add = 3
        }
    }
    
    # --- リソース ---
    resources = {
        category = planet_districts
        cost = {
            minerals = 500
        }
        upkeep = {
            energy = 2
        }
    }
    
    # --- エフェクト ---
    on_queued = { }
    on_unqueued = { }
    on_built = { }
    on_destroy = { }
    
    # --- AI設定 ---
    ai_weight = {
        weight = 0
        modifier = {
            weight = 100
            free_housing < 5
        }
    }
    ai_resource_production = {
        energy = 0
        trigger = { always = yes }
    }
}
```

## 重要なプロパティ詳細

### potential と allow の違い

| プロパティ | 建造物 | ディストリクト |
|:--|:--|:--|
| `potential = false` | GUI非表示 | GUI非表示＋削除/変換 |
| `allow = false` | GUI表示（無効化） | 建設不可＋変換 |

### 首都建造物

首都建造物には以下の特別なプロパティがあります：

```pdx
building_colony_shelter = {
    capital = yes                               # 首都として認識
    capital_tier = 1                           # ティア1首都
    can_build = no                              # 手動建設不可（自動付与）
    can_demolish = no                           # 解体不可
    can_be_ruined = no                          # 廃墟化不可
    can_be_disabled = no                        # 無効化不可
    position_priority = 0                       # 最前列配置
}
```

### 支社建造物

支社建造物は巨大企業が非巨大企業の惑星に建設できます：

```pdx
building_franchise_headquarters = {
    branch_office_building = yes                # 支社建造物として認識
    
    potential = {
        has_branch_office = yes
        branch_office_owner = { is_criminal_syndicate = no }
    }
    
    # base_cap_amountの代わりにplanet_max_branch_office_buildingsを使用
}
```

## ジョブによるリソース生産

ほとんどの建造物はリソースを直接生産せず、ジョブを提供します：

```pdx
building_alloy_foundry = {
    base_buildtime = @b2_time
    
    category = manufacturing
    
    potential = {
        NOT = { has_modifier = resort_colony }
        NOT = { has_modifier = slave_colony }
    }
    
    # 通常帝国用
    triggered_planet_modifier = {
        potential = {
            exists = owner
            owner = { is_regular_empire = yes }
        }
        modifier = {
            job_foundry_add = 2
        }
    }
    
    # 集合意識用
    triggered_planet_modifier = {
        potential = {
            exists = owner
            owner = { is_gestalt = yes }
        }
        modifier = {
            job_alloy_drone_add = 2
        }
    }
    
    resources = {
        category = planet_buildings
        cost = {
            minerals = @b2_minerals
        }
        upkeep = {
            energy = @b2_upkeep
        }
    }
    
    triggered_desc = {
        trigger = {
            exists = owner
            owner = { is_regular_empire = yes }
        }
        text = "job_foundry_effect_desc"
    }
    triggered_desc = {
        trigger = {
            exists = owner
            owner = { is_gestalt = yes }
        }
        text = "job_alloy_drone_effect_desc"
    }
}
```

## アップグレードチェーン

建造物はアップグレードチェーンを持つことができます：

```pdx
building_factory_1 = {
    base_buildtime = 360
    
    category = manufacturing
    
    resources = {
        category = planet_buildings
        cost = { minerals = 400 }
        upkeep = { energy = 3 }
    }
    
    planet_modifier = {
        job_artisan_add = 2
    }
    
    upgrades = {
        building_factory_2
    }
}

building_factory_2 = {
    base_buildtime = 480
    can_build = no                              # アップグレード専用
    
    category = manufacturing
    
    prerequisites = {
        "tech_luxuries_1"
    }
    
    resources = {
        category = planet_buildings
        cost = { minerals = 600 }
        upkeep = { energy = 5 }
    }
    
    planet_modifier = {
        job_artisan_add = 4
    }
    
    upgrades = {
        building_factory_3
    }
}
```

> [!NOTE]
> アップグレード先の建造物の `base_buildtime` と `cost` がアップグレードコストになります。`base_cap_amount` は最初の建造物にのみ設定し、上位建造物には設定しません。

## 生成されるモディファイア

`is_capped_by_modifier = yes` の建造物/ディストリクトでは、以下のモディファイアが自動生成されます：

```pdx
# 建造物/ディストリクトキー_max_add = <int>
# 例: building_clinic_max_add = 1

# 使用例
planet_modifier = {
    building_clinic_max_add = 1
}
```

## Phoenix 4.0 ゾーンシステム

Phoenix 4.0ではゾーンスロットとゾーンがディストリクトに追加されました。

### ディストリクトでのゾーン設定

```pdx
district_city = {
    default_starting_district = yes             # このセットのプライマリディストリクト
    
    zone_slots = {
        slot_city_government                    # ゾーンスロット1
        slot_city_01                            # ゾーンスロット2
        slot_city_02                            # ゾーンスロット3
    }
    
    # ...
}
```

### 建造物でのビルディングセット

```pdx
building_alloy_foundry = {
    building_sets = {
        manufacturing                           # 所属するビルディングセット
    }
    # ...
}
```

> [!IMPORTANT]
> 4.0へ移行時、以下の点に注意：
> - プライマリディストリクトに `default_starting_district = yes`を追加
> - プライマリディストリクトには3つのゾーンスロットを指定
> - すべてのディストリクトにゾーンスロットを指定
> - ディストリクトセットを5個から4個に削減
> - すべての建造物に`building_sets`を指定

詳細は [Zones modding](file:///common/zones/99_HOW_TO_ZONE.txt) を参照。

## 条件・エフェクト

### 建造物のチェック

```pdx
# 惑星スコープ
has_building = building_factory_1              # 特定の建造物が存在
num_buildings = { type = building_factory_1 value >= 2 }

# 建造物スコープ
is_ruined = yes                                # 廃墟状態か
is_under_construction = yes                    # 建設中か
```

### ディストリクトのチェック

```pdx
# 惑星スコープ
has_district = district_city                   # 特定ディストリクトが存在
num_districts = { type = district_city value >= 5 }
uses_district_set = standard                   # ディストリクトセット
free_district_slots > 0                        # 空きスロット
```

### 建造物・ディストリクトの追加

```pdx
# 惑星スコープ
add_building = building_factory_1              # 建造物を追加
remove_building = building_factory_1           # 建造物を削除

add_district = district_city                   # ディストリクトを追加
remove_district = district_city                # ディストリクトを削除
```

## ベストプラクティス

### ファイル構成

- バニラに準拠した数字プレフィックス付きファイル名を使用
- カテゴリごとにファイルを分離
- ユニークな建造物IDを使用（`building_`プレフィックス）

### 互換性

- `triggered_planet_modifier`で帝国タイプごとのジョブを指定
- `convert_to`で政体変更時の変換先を定義
- `uses_district_set`でディストリクトセットをチェック

### パフォーマンス

- `potential`は軽量なチェックを使用
- 複雑なロジックは`triggered_planet_modifier`に移動
- `inline_script`で共通ロジックを共有

### 4.0移行

- 全建造物に`building_sets`を追加
- 全ディストリクトにゾーンスロットを追加
- プライマリディストリクトに`default_starting_district = yes`を設定
