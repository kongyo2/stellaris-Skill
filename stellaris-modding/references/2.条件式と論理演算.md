# 条件式（Conditions/Triggers）と論理演算

## 概要

条件式（Conditions）またはトリガー（Triggers）は、ゲーム内の状況がtrue（真）かfalse（偽）かを評価し、単純なブール値を返す仕組みである。現在のスコープの状態を検査し、後続のエフェクト実行を制御する。

**条件式が使用される場所:**
- イベントのtriggerセクション（`mult_modifier`, `can_use_*`, `potential`, `allow`等）
- コマンドブロック内の`limit`句
- スクリプトブロック: `scripted_triggers`や`scripted_effects`として再利用可能なマクロに
- イベントのルートでpre-triggersとして（パフォーマンス最適化用、主に惑星で使用）

---

## 論理演算子

Stellarisは条件式を組み合わせるための6つの主要な論理演算子を提供する。

### AND

全ての条件がtrueの場合にtrueを返す。

```pdx
AND = {
    <condition_A>
    <condition_B>
}
```

**重要:**
- triggerや新しいスコープ内ではANDがデフォルト動作のため、明示的に指定する必要がない
- `trigger = { AND = { A B } }` は `trigger = { A B }` と等価
- **パフォーマンス最適化**: falseになりやすい条件を先に配置する

### OR

少なくとも1つの条件がtrueの場合にtrueを返す。

```pdx
OR = {
    <condition_A>
    <condition_B>
}
```

**パフォーマンス最適化**: trueになりやすい条件を先に配置する。

### NOT

条件がfalseの場合にtrue、trueの場合にfalseを返す（論理否定）。

```pdx
NOT = {
    <condition>
}
```

**制限事項:**
- NOTは**1つの条件のみ**を受け付ける
- 複数の条件を入れると動作する場合もあるが、予期しないバグが発生し、error.logにエラーが記録される
- 複数条件の否定には**NORまたはNAND**を使用すること

### NOR

全ての条件がfalseの場合にtrueを返す。`NOT = { OR = { ... } }` と等価。

```pdx
NOR = {
    <condition_A>
    <condition_B>
}
```

### NAND

少なくとも1つの条件がfalseの場合にtrueを返す（全てがtrueの場合のみfalse）。`NOT = { AND = { ... } }` と等価。

```pdx
NAND = {
    <condition_A>
    <condition_B>
}
```

### calc_true_if

指定した数以上の条件がtrueの場合にtrueを返す。`amount`は数値比較演算子をサポート。

```pdx
calc_true_if = {
    amount >= 3
    <condition_A>
    <condition_B>
    <condition_C>
    <condition_D>
    <condition_E>
}
```

## ド・モルガンの法則（De Morgan's Laws）

NORとNANDを正しく使用するために、ド・モルガンの法則を理解することが重要である。

### 法則1: NAND

```
NAND = { A B } ⟺ NOT = { AND = { A B } } ⟺ OR = { NOT = { A } NOT = { B } }
```

「AかつB」の否定は「Aでない、またはBでない」と等しい。

### 法則2: NOR

```
NOR = { A B } ⟺ NOT = { OR = { A B } } ⟺ AND = { NOT = { A } NOT = { B } }
```

「AまたはB」の否定は「Aでない、かつBでない」と等しい。

## 実用的な簡略化パターン

### AND内のNOTリストをNORに簡略化

```pdx
# 冗長な書き方
trigger = {
    NOT = { A }
    NOT = { B }
    NOT = { C }
}

# 簡略化（等価）
trigger = {
    NOR = { A B C }
}
```

### OR内のNOTリストをNANDに簡略化

```pdx
# 冗長な書き方
trigger = {
    OR = {
        NOT = { A }
        NOT = { B }
        NOT = { C }
    }
}

# 簡略化（等価）
trigger = {
    NAND = { A B C }
}
```

## XOR（排他的論理和）の実装

Stellarisには組み込みのXOR演算子がないが、以下の方法で実装できる。

### 2入力XOR

```pdx
# 方法1: OR + NANDの組み合わせ
trigger = {
    OR = { A B }
    NAND = { A B }
}

# 方法2: calc_true_if使用
trigger = {
    calc_true_if = { amount = 1 A B }
}
```

### 3入力XOR（奇数パリティ）

```pdx
trigger = {
    OR = {
        calc_true_if = { amount = 1 A B C }
        calc_true_if = { amount = 3 A B C }
    }
}
```

## 条件分岐: if / else_if / else

条件に基づいて処理を分岐させる制御構造。

```pdx
if = {
    limit = { <conditions> }
    # limit内の条件がtrueの場合に実行
}
else_if = {
    limit = { <conditions> }
    # 前のifがfalseで、このlimitがtrueの場合に実行
}
else = {
    # 全ての前提条件がfalseの場合に実行
}
```

## パフォーマンスに関する注意

- **AND**: falseになりやすい条件を先頭に配置（短絡評価による最適化）
- **OR**: trueになりやすい条件を先頭に配置
- **NOT**: 単一条件のみ使用し、複数条件にはNOR/NANDを使用
- 複雑な条件式は`scripted_triggers`に切り出して再利用性と可読性を向上
