# 条件式（Conditions/Triggers）と論理演算

## 概要

条件式（Conditions）またはトリガー（Triggers）は、ゲーム内の状況がtrue（真）かfalse（偽）かを評価し、単純なブール値を返す仕組みである。現在のスコープの状態を検査し、後続のエフェクト実行を制御する。

**条件式が使用される場所:**
- イベントのtriggerセクション（`mult_modifier`, `can_use_*`, `potential`, `allow`等）
- コマンドブロック内の`limit`句
- スクリプトブロック: `scripted_triggers`や`scripted_effects`として再利用可能なマクロに
- イベントのルートでpre-triggersとして（パフォーマンス最適化用、主に惑星で使用）

---

## 論理演算子

Stellarisは条件式を組み合わせるための6つの主要な論理演算子を提供する。

### AND

全ての条件がtrueの場合にtrueを返す。

```pdx
AND = {
    <condition_A>
    <condition_B>
}
```

**重要:**
- triggerや新しいスコープ内ではANDがデフォルト動作のため、明示的に指定する必要がない
- `trigger = { AND = { A B } }` は `trigger = { A B }` と等価
- **パフォーマンス最適化**: falseになりやすい条件を先に配置する

### OR

少なくとも1つの条件がtrueの場合にtrueを返す。

```pdx
OR = {
    <condition_A>
    <condition_B>
}
```

**パフォーマンス最適化**: trueになりやすい条件を先に配置する。

### NOT

条件がfalseの場合にtrue、trueの場合にfalseを返す（論理否定）。

```pdx
NOT = {
    <condition>
}
```

**制限事項:**
- NOTは**1つの条件のみ**を受け付ける
- 複数の条件を入れると動作する場合もあるが、予期しないバグが発生し、error.logにエラーが記録される
- 複数条件の否定には**NORまたはNAND**を使用すること

### NOR

全ての条件がfalseの場合にtrueを返す。`NOT = { OR = { ... } }` と等価。

```pdx
NOR = {
    <condition_A>
    <condition_B>
}
```

### NAND

少なくとも1つの条件がfalseの場合にtrueを返す（全てがtrueの場合のみfalse）。`NOT = { AND = { ... } }` と等価。

```pdx
NAND = {
    <condition_A>
    <condition_B>
}
```

### calc_true_if

指定した数以上の条件がtrueの場合にtrueを返す。`amount`は数値比較演算子をサポート。

```pdx
calc_true_if = {
    amount >= 3
    <condition_A>
    <condition_B>
    <condition_C>
    <condition_D>
    <condition_E>
}
```

## ド・モルガンの法則（De Morgan's Laws）

NORとNANDを正しく使用するために、ド・モルガンの法則を理解することが重要である。

### 法則1: NAND

```
NAND = { A B } ⟺ NOT = { AND = { A B } } ⟺ OR = { NOT = { A } NOT = { B } }
```

「AかつB」の否定は「Aでない、またはBでない」と等しい。

### 法則2: NOR

```
NOR = { A B } ⟺ NOT = { OR = { A B } } ⟺ AND = { NOT = { A } NOT = { B } }
```

「AまたはB」の否定は「Aでない、かつBでない」と等しい。

## 実用的な簡略化パターン

### AND内のNOTリストをNORに簡略化

```pdx
# 冗長な書き方
trigger = {
    NOT = { A }
    NOT = { B }
    NOT = { C }
}

# 簡略化（等価）
trigger = {
    NOR = { A B C }
}
```

### OR内のNOTリストをNANDに簡略化

```pdx
# 冗長な書き方
trigger = {
    OR = {
        NOT = { A }
        NOT = { B }
        NOT = { C }
    }
}

# 簡略化（等価）
trigger = {
    NAND = { A B C }
}
```

## XOR（排他的論理和）の実装

Stellarisには組み込みのXOR演算子がないが、以下の方法で実装できる。

### 2入力XOR

```pdx
# 方法1: OR + NANDの組み合わせ
trigger = {
    OR = { A B }
    NAND = { A B }
}

# 方法2: calc_true_if使用
trigger = {
    calc_true_if = { amount = 1 A B }
}
```

### 3入力XOR（奇数パリティ）

```pdx
trigger = {
    OR = {
        calc_true_if = { amount = 1 A B C }
        calc_true_if = { amount = 3 A B C }
    }
}
```

## 条件分岐: if / else_if / else

条件に基づいて処理を分岐させる制御構造。

```pdx
if = {
    limit = { <conditions> }
    # limit内の条件がtrueの場合に実行
}
else_if = {
    limit = { <conditions> }
    # 前のifがfalseで、このlimitがtrueの場合に実行
}
else = {
    # 全ての前提条件がfalseの場合に実行
}
```

## パフォーマンスに関する注意

- **AND**: falseになりやすい条件を先頭に配置（短絡評価による最適化）
- **OR**: trueになりやすい条件を先頭に配置
- **NOT**: 単一条件のみ使用し、複数条件にはNOR/NANDを使用
- 複雑な条件式は`scripted_triggers`に切り出して再利用性と可読性を向上

## スコープ（Scopes）

条件式は現在のスコープのコンテキストで評価される。スコープを切り替えることで、異なるオブジェクトの状態をチェックできる。

### 基本スコープ

| スコープ | 説明 |
|---|---|
| `root` | スクリプトが属する元のオブジェクト |
| `from` | イベント等でスクリプトをトリガーしたオブジェクト |
| `prev` | ネストチェーンの前のスコープ |
| `this` | 現在のスコープ（明示的な指定に使用） |

### オブジェクト間スコープ切り替え

| スコープ | 説明 | 使用例 |
|---|---|---|
| `owner` | 現在のオブジェクト（惑星、艦隊等）の所有国 | 惑星→国家 |
| `capital` | 現在の国家の首都惑星 | 国家→惑星 |
| `space_owner` | 星系の所有国 | 星系→国家 |
| `leader` | リーダー | 艦隊/軍→リーダー |
| `solar_system` | 惑星/艦隊の所在星系 | 惑星/艦隊→星系 |

### スコープ切り替え例

```pdx
# 惑星スコープから所有国の思想をチェック
owner = {
    has_ethic = ethic_fanatic_materialist
}

# 国家スコープから首都惑星の状態をチェック
capital = {
    has_modifier = pm_lush
}
```

## トリガーの命名規則パターン

Stellarisのトリガーは一貫した命名規則に従う。以下のパターンを理解することで、適切なトリガーを見つけやすくなる。

| プレフィックス | 意味 | 例 |
|---|---|---|
| `has_...` | 何かを持っている/存在する | `has_technology`, `has_modifier`, `has_global_flag` |
| `is_...` | 状態である | `is_ai`, `is_capital`, `is_inside_nebula` |
| `num_...` | 数値比較 | `num_districts`, `num_owned_planets`, `num_pops` |
| `any_...` | いずれかのサブオブジェクトが条件を満たす | `any_owned_pop`, `any_owned_planet` |
| `all_...` | 全てのサブオブジェクトが条件を満たす | `all_owned_pop`, `all_owned_planet` |
| `count_...` | 条件を満たすオブジェクト数 | `count_owned_pops`, `count_armies` |
| `can_...` | 何かが可能か | `can_build_army`, `can_colonize` |

## 数値比較演算子

多くのトリガーは数値比較を行う。以下の演算子が使用可能。

```pdx
# 等しい
energy = 1000

# 以上/以下
energy >= 1000
energy <= 500

# より大きい/より小さい
energy > 1000
energy < 500
```

### 変数との比較

スクリプト変数やスコープ属性との比較も可能。

```pdx
# スコープ属性との比較
check_variable = {
    which = my_variable
    value >= 10
}

# 他のオブジェクトの値との比較
num_pops >= from.num_pops
```

## スコープ反復子（Iterators）

反復子は複数のオブジェクトをループし、条件を評価する。

### any_* 反復子

少なくとも1つのオブジェクトが条件を満たす場合にtrueを返す。

```pdx
any_owned_planet = {
    has_building = building_research_lab_1
}
```

### all_* 反復子

全てのオブジェクトが条件を満たす場合にtrueを返す。

**注意**: 対象オブジェクトが0個の場合もtrueを返す（空集合に対する全称命題）。

```pdx
all_owned_pop = {
    is_enslaved = no
}
```

### count_* 反復子

条件を満たすオブジェクトの数を取得し、比較する。

```pdx
count_owned_pops = {
    limit = { is_specialist = yes }
    count >= 10
}
```

## よく使うトリガー一覧

### 国家レベル

| トリガー | 説明 |
|---|---|
| `is_ai` | AIかどうか |
| `has_technology = <tech>` | 技術を持っているか |
| `has_tradition = <tradition>` | 伝統を持っているか |
| `has_ethic = <ethic>` | 思想を持っているか |
| `has_civic = <civic>` | 国是を持っているか |
| `has_authority = <authority>` | 権威形態か |
| `has_policy_flag = <flag>` | 政策フラグがあるか |
| `has_country_flag = <flag>` | 国家フラグがあるか |

### 惑星レベル

| トリガー | 説明 |
|---|---|
| `is_capital` | 首都惑星か |
| `is_colonized` | 植民済みか |
| `has_building = <building>` | 建物があるか |
| `has_district = <district>` | 区域があるか |
| `has_modifier = <modifier>` | 惑星補正があるか |
| `free_building_slots >= <n>` | 空き建物スロット数 |
| `num_pops >= <n>` | 人口数 |

### POPレベル

| トリガー | 説明 |
|---|---|
| `is_enslaved` | 奴隷か |
| `is_being_purged` | 粛清中か |
| `has_job = <job>` | 特定の職業についているか |
| `has_citizenship_type = { type = <type> }` | 市民権の種類 |
